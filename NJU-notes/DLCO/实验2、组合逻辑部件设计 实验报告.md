# 2.2 编写8-3优先编码器
## 整体方案设计
### 顶层模块设计
根据8-3 优先级编码器原理图，设计一个由逻辑门电路构成的 8-3 优先级编码器，并将编码器输出连接到一个十六进制数码管，通过数码管的输出显示来验证和测试电路。
在本电路的实现中，通过在 AND 阵列中引入高位输入的取反信号作为低位输入的封锁条件，实现了严格的优先级判别逻辑。输出端 $O_0$ 至 $O_2$ 共同构成的二进制码与输入端最高有效索引 $i$ 满足 $Value(O_0O_1O_2) = i$ 的对应关系。通过分线器与十六进制数码管的连接，验证了在多路输入并发时，电路能准确捕捉并显示优先级最高位的编号。
### 输入输出引脚
本实验模块实现了将 8 路原始输入信号压缩编码为 3 位二进制输出的功能，并具备优先级处理机制。
#### 1. 输入引脚 (Input Pins)
该模块共有 8 个输入端，代表 8 个不同的请求源：

| 引脚名称 | 类型 | 逻辑功能描述 |
| :--- | :--- | :--- |
| **$I_0 \sim I_7$** | 输入 (1-bit) | **编码请求输入**。当某个引脚为高电平（1）时，表示该编号对应的线路发出请求。 |
| **优先级逻辑** | 核心机制 | **高位优先**。当多个输入端同时为 1 时，编码器仅对序号最大的输入进行编码。例如，若 $I_6$ 和 $I_1$ 同时有效，电路将忽略 $I_1$，输出 $I_6$ 对应的二进制码。 |

#### 2. 输出引脚 (Output Pins)
该模块输出 3 位二进制代码，用于表示当前有效请求中优先级最高者的编号：

| 引脚名称      | 类型         | 对应权重              | 逻辑功能描述                                 |
| :-------- | :--------- | :---------------- | :------------------------------------- |
| **$O_0$** | 输出 (1-bit) | **MSB ($2^2=4$)** | 编码输出的**最高位**。当选中的输入序号 $\ge 4$ 时，该位为 1。 |
| **$O_1$** | 输出 (1-bit) | **中间位 ($2^1=2$)** | 编码输出的中间位。                              |
| **$O_2$** | 输出 (1-bit) | **LSB ($2^0=1$)** | 编码输出的**最低位**。                          |

#### 3. 辅助显示组件
| 组件名称        | 类型   | 功能描述                                                                                  |
| :---------- | :--- | :------------------------------------------------------------------------------------ |
| **16进制数码管** | 输出显示 | **直观结果展示**。通过分线器（Splitter）将 $O_0, O_1, O_2$ 组合为 4 位总线信号（最高位补0），实时显示当前编码对应的十进制数值（0-7）。 |

# 2.3 4位加法器实验
## 整体方案设计
### 顶层模块设计
串联 4 个全加器子电路实现 4 位串行进位加法器，将加数、被加数和和分别连接到 16 进制数码显示管进行验证。
**补码减法原理实现：** 
根据补码运算公式：$[X - Y]_补 = [X]_补 + [-Y]_补 = [X]_补 + [\overline{Y}]_补 + 1$。
在本电路中，当控制信号 $Cin = 1$ 时：
1. 利用 $Y \oplus 1 = \overline{Y}$ 的特性，通过异或阵列将操作数 $Y$ 按位取反。
2. 将 $Cin = 1$ 直接接入最低位全加器 $FA_0$ 的进位输入端，实现了公式中的末位“+1”。
从而在不增加减法器硬件的前提下，利用现有加法器完成了补码减法运算。
### 输入输出引脚

本实验模块基于 4 个全加器（FA）级联而成，通过引入异或门阵列实现了加法和减法的电路复用。其具体引脚功能及逻辑描述如下：
#### 1. 输入引脚 (Input Pins)

| 引脚名称      | 类型   | 位宽    | 逻辑功能描述                                                                                                                                                   |
| :-------- | :--- | :---- | :------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **操作数 X** | 数据输入 | 4-bit | **被加数 / 被减数**。该 4 位总线信号通过分线器拆分，分别输入至各级全加器的 $A$ 端。                                                                                                        |
| **操作数 Y** | 数据输入 | 4-bit | **加数 / 减数**。该信号进入全加器 $B$ 端前，会经过异或门阵列处理。                                                                                                                  |
| **$Cin$** | 控制输入 | 1-bit | **模式选择与初始进位**。这是电路的核心控制信号：<br>1. **加法模式 ($Cin=0$)**：Y 原样进入加法器，初始进位为 0。<br>2. **减法模式 ($Cin=1$)**：Y 通过异或门实现**按位取反**，同时给最低位全加器提供初始进位 1，从而完成补码减法所需的“取反加一”操作。 |

#### 2. 输出引脚 (Output Pins)

| 引脚名称        | 类型   | 位宽    | 逻辑功能描述                                                                    |
| :---------- | :--- | :---- | :------------------------------------------------------------------------ |
| **SUM (F)** | 数据输出 | 4-bit | **运算结果**。由 4 位全加器计算出的和（Sum）位组合而成。在减法模式下，该输出代表 $X-Y$ 的补码结果。                |
| **$Cout$**  | 状态输出 | 1-bit | **进位/借位输出**。最高位全加器（FA3）的进位输出。在无符号加法中代表溢出；在补码减法中，若 $Cout=1$ 通常表示结果为正（无借位）。 |

#### 3. 辅助显示与组件说明
*   **16进制数码管**：用于实时直观显示操作数 $X$、$Y$ 以及运算结果 $SUM$ 的数值（0-F）。
*   **符号扩展器 (Bit Extender)**：将 1 位的控制信号 $Cin$ 扩展为 4 位，以便与 4 位操作数 $Y$ 进行位对位的异或运算。
*   **异或门阵列 (XOR Array)**：作为**可控反相器**。根据 $Cin$ 的值决定是对 $Y$ 执行“保持”还是“取反”操作。

# 2.4 汉明码校验电路
## 整体方案设计
实验目标：设计并实现一个 7 位汉明码 (7, 4) 自动检错与纠错电路。要求电路能够输入 7 位受损码（含一位错误），自动定位错误位并进行逻辑翻转修复，最终输出正确的原始数据，同时提供错误位置显示及错误标志。

### 顶层模块设计

奇偶校验是通过判断数据M中 1的个数是否发生了奇偶性变化来进行检错。 若发生奇偶变化，则故障位 $\mathbf { S } { = } \mathbf { P } ^ { \prime } \oplus \mathbf { P } ^ { \prime \prime } { = } 1$ 。
汉明码（Hamming Code，也译为海明码）的主要思想是，将数据按某种规律分成若干组，对每组进行 相应的奇偶检测，以提供多位校验信息，得到相应的故障字，根据故障字对发生的错误进行定位，并将其纠正。汉明校验码实质上就是一种多重奇偶校验码。
以下是个人对汉明码校验原理的理解。
我原本有一个4位的数据M，由公式$2^k\ge n+k+1$得到k应该取3，即设立3位校验位，然后将校验位设置在2的幂次位上与M拼成汉明码（这就是Input），原本数据信息为$M_4M_3M_2M_1$，则码字排列为$P_1P_2M_1P_3M_2M_3M_4$，将M分为了3组，$P_i$由第i位为1的所有数（除$P_i$以外）做异或运算生成。然后错误位$S_i$由第i位为1的所有数做异或运算生成，3个错误位拼接成的错误码直接指向出错的数在汉明码中的位置。则翻转错误码对应的数即可完成修正。

本设计采用 **“校验-译码-纠错”** 的三级流水结构，顶层电路主要由以下逻辑块组成：
1.  **输入接口层**：接收 7 位汉明码（$D_1 \sim D_7$）。
2.  **故障字生成层**：由三组 4 位奇偶校验器组成，产生 3 位故障字（Syndrome）。
3.  **译码定位层**：利用 74x138 (3-8 译码器) 将 3 位故障字转换为 7 路选通信号。
4.  **修正输出层**：通过异或门阵列对错误位执行按位取反，并汇总输出。

#### 子模块详细描述
*   **4位奇偶校验器 (Parity Checker)**：
    *   **功能**：对输入的 4 位数据执行异或运算（EVEN 端口）。
    *   **在本实验中的作用**：按照汉明码分组规则，计算三组校验方程的奇偶性，输出结果构成故障字。
*   **74x138 译码器**：
    *   **功能**：将 3 位二进制地址转换为 8 路低电平有效信号。
    *   **在本实验中的作用**：作为“指证码”执行器，当故障字不为 0 时，精准激活对应错误位的纠错信号。
*   **纠错异或阵列 (XOR Array)**：
    *   **功能**：利用 $A \oplus 1 = \bar{A}$ 且 $A \oplus 0 = A$ 的特性。
    *   **在本实验中的作用**：将原始输入与译码器的输出相连。仅在被译码器“点名”的错误位上执行取反操作。

#### 数据与控制信号传输通道
1.  **数据流**：`Input` $\rightarrow$ `Splitter`（拆分） $\rightarrow$ 分为两路。
    *   **路径 A**：进入校验器计算故障字。
    *   **路径 B**：直接送往末端异或门等待修正。
2.  **控制流**：`故障字` $\rightarrow$ `3-8 译码器` $\rightarrow$ `选通控制线` $\rightarrow$ `控制异或门翻转`。
3.  **反馈流**：故障字经过 NOR 逻辑判断是否全 0，产生 `NoError` 全局标志信号。

### 输入输出引脚

| 引脚名称        | 类型  | 位宽    | 描述                         |
| :---------- | :-- | :---- | :------------------------- |
| **Input**   | 输入  | 7-bit | 待检测的 7 位汉明码总线信号            |
| **Output**  | 输出  | 7-bit | 纠错后的 7 位结果总线信号             |
