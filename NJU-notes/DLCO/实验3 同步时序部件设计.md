# 3.1 计数器实验
## 整体方案设计
### 顶层模块设计
根据表 3.1 给出的功能表和图 3.1 所示电路原理图构建 4 位二进制同步计数器电路。
### 输入输出引脚

本电路模块（CNTR4U）是一个典型的中规模集成时序逻辑器件，其引脚设计遵循优先级控制原则，所有状态转换均在时钟边沿同步发生。
#### 1. 输入控制引脚 (Input Control Pins)

| 引脚名称    | 功能类型 | 触发/有效电平   | 详细描述                                                                              |
| :------ | :--- | :-------- | :-------------------------------------------------------------------------------- |
| **CLK** | 时钟输入 | **上升沿触发** | 电路的主时钟。所有状态（计数、置数、清零）的改变均在 CLK 的上升沿（0 $\to$ 1）瞬间执行。                               |
| **CLR** | 同步清零 | 高电平有效     | **最高优先级**。当 CLR=1 时，忽略其他输入，在下一个时钟上升沿将 $Q_3 \sim Q_0$ 全部置为 0。                      |
| **LD**  | 同步置数 | 高电平有效     | **第二优先级**。当 CLR=0 且 LD=1 时，在时钟上升沿将数据端 $D_3 \sim D_0$ 的数值并行载入触发器。                  |
| **ENP** | 计数使能 | 高电平有效     | **普通使能**。控制计数器是否进入计数状态。                                                           |
| **ENT** | 计数使能 | 高电平有效     | **进位使能**。除控制计数外，还直接控制 $RCO$ 的输出。$ENP$ 与 $ENT$ 必须**同时为 1** 且高优先级信号无效时，电路才执行加 1 计数。 |

#### 2. 数据输入引脚 (Data Input Pins)

| 引脚名称 | 类型 | 位宽 | 详细描述 |
| :--- | :--- | :--- | :--- |
| **$D_0 \sim D_3$** | 并行输入 | 4-bit | 用于同步置数功能的数据源。$D_0$ 为最低位，$D_3$ 为最高位。 |

#### 3. 输出引脚 (Output Pins)

| 引脚名称               | 类型   | 位宽    | 详细描述                                                                                                                 |
| :----------------- | :--- | :---- | :------------------------------------------------------------------------------------------------------------------- |
| **$Q_0 \sim Q_3$** | 状态输出 | 4-bit | 计数器当前计数值的二进制表示。$Q_0$ 是 LSB（权值 1），$Q_3$ 是 MSB（权值 8）。                                                                  |
| **RCO**            | 进位输出 | 1-bit | **进位/溢出标志**。当计数值达到最大值 15 (1111) 且 $ENT=1$ 时，该引脚输出高电平，用于级联下一片计数器。 $RCO = Q_0 \cdot Q_1 \cdot Q_2 \cdot Q_3 \cdot ENT$ |
|                    |      |       |                                                                                                                      |
控制逻辑优先级排序为：同步清零 ($CLR$) > 同步置数 ($LD$) > 计数使能 ($ENP \cdot ENT$) > 保持
**时序特性补充说明：**  
经实际仿真测试，本实验所采用的 4 位二进制同步计数器为 **上升沿触发**。  
观察原理图（图 3.1）发现，虽然内部 D 触发器采用带圆圈的下降沿触发设计，但由于时钟输入端（CLK）前置了一级非门，经过双重逻辑取反后，系统整体呈现为上升沿同步特性。这一设计符合标准 74LS161 系列集成电路的时序逻辑规范，能够有效利用时钟正跳变沿稳定捕获数据，避免竞争冒险。

# 3.5 设计数字时钟
## 整体方案设计
### 顶层模块设计
在 6 个 7 段数码管上显示数字时钟时分秒，当计时到 23:59:59 后进入 00:00:00，时分秒之间用小数点分隔；整点时点亮三色 LED 灯组件，按照格雷码赋值，每秒显示一种颜色，持续显示 10 秒；通过 8421BCD 码设置初始时间，在载入时，如果初始时间数值超出实际范围，则报输入错，且不能被载入。
利用同步计数器电路实现 0~9 十进制计数器、0~59 六十进制计数器和 0~23 二十四进制计数器。当时分秒的低位计数到 9 时，高位计数加 1，低位清零。当秒数计数到 59 后秒数清零，分钟计数加 1；当分钟计数达到 59 且秒数也到达 59 秒时，分钟清零，小时计数加 1；当小时计数到 23，且分钟和秒数都是 59 时，小时数清零。当载入信号 LD=1 有效时，检查设置时间数值：小时高低位 HH1 和 HH0、分钟高低位 MM1 和 MM0、秒数高低位 SS1 和SS0，如果在有效数据范围之内，则载入时钟初值，否则 INErr 赋值 1。当计时到 59 分 59 秒进入整点时，三色 RGB LED 灯按照格雷码（蓝色为高位、红色为低位）的顺序轮流点亮三色灯（红色、黄色、绿色、青色、白色、品红、蓝色、黑色、红色、黄色），并持续 10 个时钟周期。需要独立设计电路进入计数状态，设置开始计数标志位和结束计数标志位，并实现格雷码的转换。

然后发现我的电路在进入整点时亮灯顺序有误，观察后注意到格雷码连错了，调整连线即可，
同时，在测试结果中，可以看到在 `0c` 行（即 $00:00:00$ 整点那一秒），`HH/MM/SS` 已经全部清零了，但 `LED_R` 还是 **0**，直到下一秒 `0d` 才变红。
### 为什么会慢一拍？
1.  **触发逻辑的链条**：
    *   **T=0（59:59）**：检测电路（那个大与门）发现了整点，发出一个 `START` 信号。
    *   **T=1（00:00）**：在时钟上升沿，`START` 信号被 **D 触发器** 捕捉，`LED_EN` 标志位变 **1**。
    *   **注意**：此时 `LED_EN` 刚刚变 1，它连在计数器的 `ENP/ENT` 上。这意味着计数器在这一秒刚被“准许工作”，它要等到**再下一个**时钟上升沿（T=2）才会从 0 变到 1。
2.  **结果**：整点那一秒，计数器还是 0，所以 LED 是黑的。
###  解决方案：利用“同步预置 (LD)”实现抢拍
既然时钟沿是整齐划一的，我们不能让它提前，但我们可以让计数器在“被唤醒”的那一瞬间**直接跳到第 1 秒的状态**，而不是从 0 开始慢慢数。直接把 LED 计数器的输入引脚 **$D_3 D_2 D_1 D_0$** 设为常量 **`0001`**（二进制 1）即可。
