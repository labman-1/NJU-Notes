# 4.1 4位先行进位加法器CLA
## 整体方案设计
目标：本实验旨在设计并实现一个 **4位先行进位加法器（CLA4）**。通过构建先行进位部件（CLU），消除串行进位加法器中进位信号逐级传递产生的累积延迟，从而提高运算部件的整体执行速度。
### 1.2 顶层设计模块图
本系统（CLA4）采用自底向上的层级设计，顶层模块由以下部分级联而成：
*   **输入模块**：接收 2 个 4 位操作数 $X, Y$ 及初始进位 $C_0$。
*   **计算核心**：由 **4 个改进型全加器（FA）** 和 **1 个 4 位先行进位部件（CLU）** 组成。
*   **输出模块**：输出 4 位和数 $F$、进位 $C_{out}$，以及成组进位生成信号 $G_g$ 和传递信号 $P_g$（用于更高位扩展）。

### 1.3 子模块详细描述
1.  **改进型全加器 (FA)**：
    *   **输入**：$A, B, C_{in}$。
    *   **输出**：和数 $F$、进位生成函数 $G$ ($A \cdot B$)、进位传递函数 $P$ ($A + B$)。
    *   **作用**：不再在内部计算进位输出，而是提取本位的进位特征交给 CLU。
2.  **4位先行进位部件 (CLU)**：
    *   **输入**：4对 $P_i, G_i$ ($i=0 \sim 3$) 及初始进位 $C_0$。
    *   **输出**：$C_1 \sim C_4$，成组进位生成 $G_g$，成组进位传递 $P_g$。
    *   **作用**：基于布尔方程 $(4-1)$，通过两级门电路并行计算出所有位所需的进位信号。

### 1.4 引脚定义与传输通道
*   **数据通道**：$X, Y \xrightarrow{Splitter} FA \xrightarrow{P_i, G_i} CLU \xrightarrow{C_i} FA \xrightarrow{Sum} F$。
*   **关键路径**：进位产生不再依赖 $C_{i-1}$，而是由 $C_0, P_i, G_i$ 经过两级与-或逻辑直接生成。

# 4.2 16位两级先行进位加法器实验
## 整体方案设计
#### 1. 实验目标
基于 4 位先行进位加法器（CLA4）和 4 位先行进位部件（CLU），构建一个 **16 位两级先行进位加法器**。该设计旨在实现“组内并行”与“组间并行”的双重优化，从而极大地压缩 16 位加法运算的进位传递延迟。

#### 2. 顶层设计模块图描述
顶层模块 `Adder16` 采用了典型的**层次化（Hierarchy）** 设计架构。其逻辑结构由四个  **CLA4（4位先行进位加法器）**  和一个 **CLU（组间先行进位部件）** 构成。

*   **设计逻辑**：将 16 位操作数分为 4 组（每组 4 位）。
*   **第一级（组内）**：每个 CLA4 模块内部并行计算组内的 4 位加法及进位，并向外提供本组的成组进位产生信号（$G_g$）和成组进位传递信号（$P_g$）。
*   **第二级（组间）**：顶层的 CLU 部件接收来自 4 个 CLA4 模块的 $P_g$ 和 $G_g$ 信号，以及初始进位 $C_0$，并行计算出各组所需的进位输入信号 $C_4, C_8, C_{12}$ 以及最终进位 $C_{16}$。

#### 3. 子模块详细描述
1.  **4位先行进位加法器 (CLA4)**：
    *   **核心功能**：实现 4 位二进制加法。
    *   **关键输出**：除了和数 $S[3:0]$，必须输出 $G_g$（Group Generate）和 $P_g$（Group Propagate）。这两个信号描述了该 4 位块作为一个整体对进位的产生和传递特性，是实现第二级并行的基础。
2.  **4位先行进位部件 (CLU)**：
    *   **核心功能**：作为组间进位管理器。
    *   **逻辑行为**：它并不处理加法，而是处理 $P/G$ 逻辑。它将 4 个 CLA4 视为 4 个“超级全加器”，根据它们的成组特征信号，利用先行进位公式直接生成组与组之间的进位。

#### 4. 引脚定义与信号传输通道
##### (1) 外部引脚定义
| 引脚名称 | 类型 | 位宽 | 描述 |
| :--- | :--- | :--- | :--- |
| **X** | 输入 | 16-bit | 被加数总线 |
| **Y** | 输入 | 16-bit | 加数总线 |
| **C0** | 输入 | 1-bit | 初始进位输入 |
| **S** | 输出 | 16-bit | 16 位加法运算和数结果 |
| **C16** | 输出 | 1-bit | 最高位进位输出（溢出标志） |
| **Pg / Gg** | 输出 | 1-bit | 16 位整体的成组进位传递/产生信号（用于 64 位扩展） |

##### (2) 内部传输通道逻辑
1.  **数据分发**：16 位总线 $X, Y$ 经由 **分线器（Splitter）** 拆分为四组 4 位信号，分别送入四个 `CLA4` 模块的 $A$ 和 $B$ 端。
2.  **进位预测流**：四个 `CLA4` 产生的四对 $(P_{gi}, G_{gi})$ 汇总送入中央 `CLU`。
3.  **进位反馈流**：`CLU` 瞬间计算出组间进位，将 $C_4$ 给第二组， $C_8$ 给第三组， $C_{12}$ 给第四组。
4.  **结果汇总**：四个 `CLA4` 算出的四组 4 位 $S$ 信号经由 **分线器** 合并为 16 位总线 $S[15:0]$ 输出。

# 4.3 32位快速加法器实验
## 整体方案设计
#### 1. 实验目标
本实验旨在利用两个 **16位两级先行进位加法器（Adder16）** 串行级联，构建一个完整的 **32位快速加法器（Adder32）**。同时，根据算术运算逻辑，设计并实现进位（CF）、溢出（OF）、符号（SF）和零（ZF）四个状态标志位生成电路，为后续 ALU 的构建奠定基础。
#### 2. 顶层设计模块图描述
顶层模块 `Adder32` 采用典型的级联与状态监测结构：
*   **级联结构**：将 32 位操作数拆分为高、低两个 16 位部分。低 16 位加法器（Adder16_Low）处理 $X[15:0]$ 和 $Y[15:0]$，其进位输出 $C_{16}$ 直接作为高 16 位加法器（Adder16_High）的进位输入。
*   **标志位逻辑块**：分布在加法器核心周边。它通过提取操作数的最高位（符号位）、最终进位信号以及 32 位运算结果，实时计算当前运算的状态特征。

#### 3. 子模块详细描述
1.  **16位两级先行进位加法器 (Adder16)**：
    *   本实验的核心运算单元。内部包含 4 组 CLA4 模块和一个组间 CLU，能快速生成 16 位和数。
    *   **Low 模块**：处理 $0 \sim 15$ 位。
    *   **High 模块**：处理 $16 \sim 31$ 位，其结果的最高位 $F_{31}$ 直接影响标志位。
2.  **标志位生成逻辑 (Flag Logic)**：
    *   **OF（溢出）**：采用符号位判别法。利用公式 $OF = \overline{A_{31}}\overline{B_{31}}F_{31} + A_{31}B_{31}\overline{F_{31}}$。逻辑意义为：两个正数相加得负数，或两个负数相加得正数，即为溢出。
    *   **SF（符号）**：直接取结果的最高位 $F_{31}$。
    *   **CF（进位）**：反映无符号数溢出。实现逻辑为 $CF = C_{out} \oplus C_{in}$（注：此处通常配合减法逻辑设计，加法时主要由 $C_{32}$ 决定）。
    *   **ZF（零）**：利用多级或非门树实现。为了避免单门扇入过多（32个输入），本实验采用分层或运算（每 8 位或 16 位一组），最后通过总或非逻辑输出。

#### 4. 引脚定义与信号传输通道
##### (1) 引脚定义
| 引脚名称 | 类型 | 位宽 | 功能描述 |
| :--- | :--- | :--- | :--- |
| **X / Y** | 输入 | 32-bit | 32 位操作数总线 A 和 B |
| **Cin** | 输入 | 1-bit | 初始进位信号 |
| **Result (S)**| 输出 | 32-bit | 32 位加法运算结果总线 |
| **Cout** | 输出 | 1-bit | 最高位进位输出（即 $C_{32}$） |
| **CF / OF** | 输出 | 1-bit | 进位标志 / 补码溢出标志 |
| **SF / ZF** | 输出 | 1-bit | 符号标志 / 零结果标志 |

##### (2) 信号传输通道逻辑
1.  **数据流**：32 位 $X, Y$ 输入 $\xrightarrow{Splitter}$ 低 16 位给 `Adder16_Low`，高 16 位给 `Adder16_High`。
2.  **进位链**：$Cin \to \text{Adder16\_Low} \to C_{16} \to \text{Adder16\_High} \to C_{32}(Cout)$。
3.  **状态监测流**：
    *   将 $A_{31}, B_{31}, F_{31}$ 引出至异或/与或门阵列计算 **OF**。
    *   将 32 位 $F$ 结果总线通过 **Splitter** 分解后，送入多级 NOR 结构计算 **ZF**。

# 4.4 32位桶形移位器实验
## 整体方案设计
#### 1. 实验目标
设计并实现一个 32 位多功能桶形移位器（Barrel Shifter）。该电路要求能够在一个时钟周期内完成 32 位数据的逻辑左/右移、算术右移及循环左移运算，旨在掌握基于多路选择器（MUX）的对数级联设计思想，优化大位宽数据的移位速度。
#### 2. 顶层设计模块图描述
顶层模块 `Shifter32` 采用 **5 级流水级联架构**。每一级由一组 32 位的 2 选 1 多路选择器阵列构成，各级根据移位量 `shamt[4:0]` 的对应位权决定是否执行位移。

*   **级联逻辑**：
    *   **第一级**：受 `shamt[0]` 控制，决定是否移动 $2^0 = 1$ 位。
    *   **第二级**：受 `shamt[1]` 控制，决定是否移动 $2^1 = 2$ 位。
    *   **第三级**：受 `shamt[2]` 控制，决定是否移动 $2^2 = 4$ 位。
    *   **第四级**：受 `shamt[3]` 控制，决定是否移动 $2^3 = 8$ 位。
    *   **第五级**：受 `shamt[4]` 控制，决定是否移动 $2^4 = 16$ 位。
*   **控制矩阵**：通过 `L/R` 和 `A/L` 两个信号的组合，控制空位填充逻辑（补 0、补符号位或循环拼接）。
#### 3. 子模块详细描述
1.  **多路选择器阵列 (MUX Array)**：
    *   直接使用 Logisim 中的 32 位宽 MUX。
    *   **0 端**：接上一级原始输出（保持）。
    *   **1 端**：接上一级偏移后的数据（执行位移）。
2.  **数据拼接/填充模块 (Bit Manipulation)**：
    *   利用 **分线器（Splitter）** 重新排列总线顺序。
    *   **逻辑移位**：空位填入常量 0。
    *   **算术移位**：空位通过 **位扩展器（Bit Extender）** 或直接引入符号位 $Din[31]$ 进行填充。
    *   **循环移位**：将移出的高/低位循环拼接到另一端。

#### 4. 引脚定义与信号传输通道
##### (1) 外部引脚定义
| 引脚名称 | 类型 | 位宽 | 逻辑功能描述 |
| :--- | :--- | :--- | :--- |
| **Din** | 输入 | 32-bit | 待处理的原始 32 位数据 |
| **shamt** | 输入 | 5-bit | 移位量，范围 0-31 |
| **L/R** | 输入 | 1-bit | 方向控制（与 A/L 配合） |
| **A/L** | 输入 | 1-bit | 类型控制（与 L/R 配合） |
| **Dout** | 输出 | 32-bit | 移位运算后的 32 位结果 |

##### (2) 传输通道与控制逻辑约定
根据题目提示，控制逻辑定义如下：
*   **00 (逻辑右移)**：高位补 0，低位丢弃。
*   **01 (算术右移)**：高位补符号位（$Din[31]$），低位丢弃。
*   **10 (逻辑左移)**：低位补 0，高位丢弃。
*   **11 (循环左移)**：高位移出的位重新从低位补入。

# 4.5  RV32I ALU
## 整体方案设计
#### 1. 实验目标
设计并实现一个符合 RV32I 指令集规范的 32 位算术逻辑部件（ALU）。该部件需支持算术加减、逻辑运算（与/或/异或）、多种模式移位及大小比较置位功能，并通过一个 4 位的控制信号 `ALUctr` 动态切换运算类型。

#### 2. 顶层设计模块图描述
顶层模块 `ALU` 采用 **“控制通路与数据通路解耦”** 的设计架构：
*   **控制中心**：由 `ALUctrl` 子模块负责。它接收 4 位总控信号，解码产生 7 个内部细分控制信号，指挥各运算部件的工作及结果选通。
*   **数据通路**：采用并行架构。操作数 A 和 B 同时送入加法器、移位器及逻辑门阵列。各部件并行计算出可能的结果，最后通过一个巨大的 8 选 1 多路选择器（MUX）根据控制信号选出最终的 `Result`。

#### 3. 子模块详细描述
1.  **ALU 操作控制部件 (ALUctrl)**：
    *   **核心功能**：指令译码。
    *   **逻辑实现**：基于表 4.1 的最小项表达式，将 `ALUctr[3:0]` 映射为加减控制（SUBctr）、符号比较控制（SIGctr）、移位模式（ALctr/LRctr）及输出选通（OPctr）。
2.  **32位快速加法器 (Adder32)**：
    *   **核心功能**：实现 $A + B$ 或 $A - B$（通过 $B \oplus SUBctr$ 和 $C_{in}=SUBctr$ 实现减法）。
    *   **标志位作用**：输出的 CF, OF, SF 用于后续的“小于比较（Less）”逻辑判断。
3.  **32位桶形移位器 (Shifter32)**：
    *   **核心功能**：根据 B 的低 5 位（shamt）执行逻辑左/右移、算术右移或循环左移。
4.  **比较逻辑与选通阵列**：
    *   **Less 逻辑**：根据加法器标志位，实现 $(SF \oplus OF)$（带符号比较）或 $CF$（无符号比较）的选通。
    *   **逻辑阵列**：独立的 32 位按位与（AND）、按位或（OR）、按位异或（XOR）门阵列。

#### 4. 引脚定义与信号传输通道
| 引脚名称 | 类型 | 位宽 | 逻辑功能描述 |
| :--- | :--- | :--- | :--- |
| **A / B** | 输入 | 32-bit | 源操作数总线 |
| **ALUctr** | 输入 | 4-bit | 运算类型总控制码 |
| **Result** | 输出 | 32-bit | 运算结果输出总线 |
| **Zero** | 输出 | 1-bit | 零标志位，若 Result 为全 0 则输出 1 |

## 4.
### 分析：
独立比较器：使用专门的逻辑阵列直接对比A和B的位是否相同，输出0或1.
减法标志位：执行A-B，若为有符号数，则通过检查$SF\oplus OF$、ZF
来判断A与B的大小；若为无符号数则看CF和ZF即可。
### 特性：
1. 逻辑延迟差异：从电路复杂度来看，独立的比较器在最差情况下的逻辑级数不会超过32位加法器，这是因为比较器无需产生和，还可以构建类似优先级编码器的树形电路，可类比超前进位逻辑，不会超过加法器的关键路径。
2. 硬件开销与复用：从工程复用的角度来看，复用ALU的减法器来处理比较运算无需额外放置专门的逻辑阵列来比较大小，面积更优，指令集紧凑型更好。
3. 时序与毛刺：在“减法标志位”方案中，SF和OF的稳定时间受加法器进位的影响，在`slt`指令执行周期内，若加法器进位尚未稳定，此时读取标志位进行“置位”操作会有毛刺风险。
