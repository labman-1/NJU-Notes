# 5.1 只读存储器实验
## 整体方案设计

#### 1. 顶层设计模块图描述
本实验的顶层模块逻辑架构可以概括为 **“输入校验 -> 地址变换 -> 并行检索 -> 点阵驱动”**。

*   **设计思想**：由于字库文件是线性的（每个字符占 16 字节），而 LED 点阵需要同时接收 16 行数据。我们采用**存储器阵列** 的思想，将 16 个加载了相同字库内容的 ROM 并联。通过给每个 ROM 施加一个固定的地址偏移量（0~15），实现单次寻址即可同步取出该字符的完整 16 字节点阵。

#### 2. 子模块详细描述
1.  **范围检查器**：
    *   利用比较器判断输入的 ASCII 码。若码值 $< 0x20$ 或 $> 0x7F$，则判定为不可见字符。
2.  **地址计算单元**：
    *   **减法器**：执行 $Base = Code - 0x20$，将 ASCII 码映射到从 0 开始的字符索引。
    *   **左移器**：将差值左移 4 位（相当于 $\times 16$），得到该字符在字库中的绝对起始地址。
3.  **ROM 存储阵列**：
    *   由 16 个 11 位地址、8 位数据的只读存储器组成。
    *   所有 ROM 加载相同的 `ascii8-16.zk` 镜像。
4.  **LED 驱动模块**：
    *   将 16 个 ROM 的 8 位输出分别对应连接至 LED 点阵的 16 行输入端。

#### 3. 引脚定义与传输通道
*   **输入引脚**：`Code` (7-bit ASCII 码)。
*   **输出引脚**：`row1 ~ row16` (8-bit 数据线)，`CodeErr` (1-bit 错误标志)。
*   **传输通道**：
    *   **地址总线**：11 位，由计算单元产生的起始地址经过加法器链（$+0, +1, \dots, +15$）分发至各 ROM。
    *   **数据总线**：16 条 8 位独立通道，将字形信息并行传给点阵。

# 5.2数据存储器实验
## 整体方案设计
#### 1. 顶层设计模块图描述
本实验的顶层模块名为 `RAM_Module`。设计采用了 **“分块存储+ 路由选通”** 的架构。

*   **设计思想**：将 256KB 空间拆分为 4 个独立的 8 位存储体（$RAM_3 \sim RAM_0$）。这种设计允许我们通过控制每一片的片选信号（$SEL_i$），实现对任意字节、半字或整字的精准存取，而无需复杂的位移运算。
*   **层次结构**：
    1.  **地址与对齐检查逻辑**：分析 $MemOp$ 和地址低 2 位，生成片选信号及对齐错误标志。
    2.  **写数据路由逻辑**：将 32 位输入 $Din$ 的对应字节“路由”到目标 RAM 的输入端。
    3.  **存储阵列**：4 片 $64K \times 8$-bit RAM，通过地址线 $Addr[17:2]$ 并行寻址。
    4.  **读数据选通与扩展逻辑**：从 4 片 RAM 中提取目标数据，并根据 $MemOp$ 进行 0 扩展或符号扩展，最终拼装为 32 位。

#### 2. 子模块详细描述
1.  **控制信号译码器 (Control Decoder)**：
    *   输入 $MemOp[2:0]$ 和 $Addr[1:0]$。依据表 5.2 逻辑，输出 $SEL_{3:0}$。若访问不满足自然对齐（如地址为 1 时读取字），则令 $AlignErr=1$。
2.  **写通路多路开关**：
    *   根据 $Addr[1:0]$，决定 $Din$ 的低字节是给 $RAM_0, RAM_1, RAM_2$ 还是 $RAM_3$。
3.  **读通路选择与扩展器 (Read Post-Processor)**：
    *   利用多路选择器提取目标字节/半字。
    *   **符号扩展单元**：根据 $MemOp[2]$ 判断，若为有符号加载（`lb`, `lh`），则将提取数据的最高位复制到高 24/16 位。

#### 3. 引脚定义与传输通道
*   **主要输入**：`Addr[17:0]`（18位地址）、`Din[31:0]`、`MemOp[2:0]`、`WE`（写使能）。
*   **主要输出**：`Dout[31:0]`、`AlignErr`（对齐错误）。
*   **数据通路**：$Din \xrightarrow{分线器} \text{写选择逻辑} \xrightarrow{8-bit} RAM[0:3] \xrightarrow{8-bit} \text{读选通逻辑} \xrightarrow{位扩展} Dout$。

# 5.3 取指令部件实验 
## 整体方案设计
取指令部件（Instruction Fetch Unit, IFU）是单周期 CPU 数据通路的最前端，负责在每个时钟周期的上升沿根据当前程序计数器（PC）的值，从指令存储器（Instruction ROM）中取出对应的 32 位 RISC-V 指令，并根据指令类型与跳转控制逻辑实时计算出下一条指令的地址。

---

## 一、 顶层数据通路与控制逻辑设计框图

IFU 的顶层结构由**程序计数器（PC）**、**指令存储器（Instruction ROM）**、**地址加法计算链**以及**多路选择器（MUX）**构成。

```text
                     +---------------------------------------+
                     |             下地址逻辑                |
                     |   (Next PC Generation Logic)          |
                     |                                       |
                     |     +-----+                           |
 BusA (Rs1) -------> | 1 ->|     |                           |
                     |     | MUX |-+                         |
  PC (Current) ----> | 0 ->|     | | (NPCASrc)               |
                     |     +-----+ |                         |
                     |             v                         |
                     |          +-----+                      |
                     |          | Add | ----+ (Next PC)      |
                     |          +-----+     |                |
                     |             ^        |                |
                     |     +-----+ |        |                |
   Imm ------------> | 1 ->|     | |        |                |
                     |     | MUX |-+        |                |
     4 ------------> | 0 ->|     |   (NPCBSrc)               |
                     |     +-----+          |                |
                     +----------------------|----------------+
                                            |
   Reset (高电平有效)                       |
     |                                      |
     v                                      v
  +-------------------------------------------+
  |               PC 寄存器 (32-bit)          |
  |  - 若 Reset=1: PC <- Initial Address      |
  |  - 若 Reset=0: PC <- Next PC (CLK上升沿)   |
  +-------------------------------------------+
     |
     +-------> PC 物理地址输出 (32-bit)
     |
     v [17:2]
  +-------------------------------------------+
  |       指令存储器 ROM (256KB, 16位地址)    | --> IR 指令输出 (32-bit)
  +-------------------------------------------+
```

---

## 二、 子模块详细描述与技术实现

取指令部件（IFU）内部由三个核心功能子模块级联构成：

### 1. 程序计数器模块 (Program Counter, PC)
*   **功能描述**：用于保存当前正在执行指令的 32 位物理内存地址。在系统复位信号（Reset）有效时，PC 异步或同步强制初始化为指定的程序入口首地址（`Initial Address`）；在正常运行状态下，每个时钟信号（CLK）的上升沿到来时，采样并锁存下一条指令的计算地址（`Next PC`）。
*   **时序特征**：上升沿触发，具备高电平同步复位功能。

### 2. 指令存储器模块 (Instruction ROM)
*   **功能描述**：用于存储编译好的 32 位定长 RISC-V 机器指令。
*   **编址映射（重点）**：
    *   RISC-V 架构采用**字节编址**（每个地址对应 1 字节），32 位指令占 4 字节，故合法指令地址的最低两位 `PC[1:0]` 始终为 `00`。
    *   Logisim 的 ROM 组件是**字编址**（每个地址对应 4 字节）。
    *   **地址转换关系**：为了将 32 位的 PC 地址映射到 16 位地址线的 ROM，需要丢弃 PC 的最低两位 `PC[1:0]`（舍弃字节偏移）以及高 14 位 `PC[31:18]`（超出 256KB 寻址范围），仅提取 **`PC[17:2]`** 作为 ROM 的 16 位输入地址。

### 3. 下地址生成逻辑 (Next PC Logic)
*   **功能描述**：根据控制信号 `NPCASrc` 和 `NPCBSrc` 的配置，实时计算 `Next PC`，以支持顺序执行、条件分支跳转（b-type）和无条件跳转（jal, jalr）。
*   **逻辑计算公式**：
    *   当 `NPCASrc = 0, NPCBSrc = 0` 时（顺序执行）：$\text{Next PC} = \text{PC} + 4$
    *   当 `NPCASrc = 0, NPCBSrc = 1` 时（JAL 或 Branch 成功）：$\text{Next PC} = \text{PC} + \text{Imm}$
    *   当 `NPCASrc = 1, NPCBSrc = 1` 时（JALR 指令）：$\text{Next PC} = \text{BusA(Rs1)} + \text{Imm}$

## 三、 接口信号与引脚定义

| 信号名称 | 方向 | 位宽 | 物理属性 | 功能定义 |
| :--- | :--- | :--- | :--- | :--- |
| **CLK** | 输入 | 1 | 时钟信号 | 系统主时钟，**上升沿**触发 PC 的更新。 |
| **Reset** | 输入 | 1 | 复位控制 | **高电平有效**。为 1 时，在 CLK 到来时将 `Initial Address` 装入 PC。 |
| **Initial Address**| 输入 | 32 | 数据总线 | 系统的程序入口地址（例如 `0x00000400`）。 |
| **Imm** | 输入 | 32 | 数据总线 | 由译码部件（IDU）扩展输出的 32 位立即数。 |
| **BusA** | 输入 | 32 | 数据总线 | 源寄存器 1（rs1）读取出的 32 位数据（用于 jalr）。 |
| **NPCASrc** | 输入 | 1 | 控制信号 | 选择下地址加法器输入端 A：`0` 代表当前 PC，`1` 代表 BusA。 |
| **NPCBSrc** | 输入 | 1 | 控制信号 | 选择下地址加法器输入端 B：`0` 代表常量 4，`1` 代表立即数 Imm。 |
| **PC** | 输出 | 32 | 数据总线 | 当前正在执行的 32 位指令物理地址。 |
| **IR** | 输出 | 32 | 数据总线 | 从 Instruction ROM 中取出的当前 32 位机器指令。 |

## 四、 数据及控制信号传输通道流程分析

### 1. 顺序取指通路（PC = PC + 4）
*   **控制状态**：`NPCASrc = 0`，`NPCBSrc = 0`
*   **数据流向**：
    1.  `PC` 寄存器的当前值同时输出至 `ROM` 的地址端（经过截断为 `PC[17:2]`）和加法器 A 输入选通 MUX。
    2.  加法器 B 选通 MUX 选中常量 `4`。
    3.  专用加法器计算出 $\text{PC} + 4$，送回 PC 寄存器的输入端。
    4.  时钟上升沿到来，PC 载入新值 $\text{PC} + 4$。

### 2. 无条件/有条件相对跳转通路（PC = PC + Imm）
*   **控制状态**：`NPCASrc = 0`，`NPCBSrc = 1`
*   **数据流向**：
    1.  加法器 A 端 MUX 选中当前 `PC`。
    2.  加法器 B 端 MUX 选通当前译码产生的立即数 `Imm`。
    3.  专用加法器计算出 $\text{PC} + \text{Imm}$ 并送入 PC 寄存器输入端，完成跳转地址的预备。

### 3. 寄存器间接跳转通路（PC = BusA + Imm）
*   **控制状态**：`NPCASrc = 1`，`NPCBSrc = 1`
*   **数据流向**：
    1.  加法器 A 端 MUX 切换至 `1`，读取寄存器堆输出的 `BusA`。
    2.  加法器 B 端 MUX 切换至 `1`，读取立即数 `Imm`。
    3.  专用加法器计算出 $\text{BusA} + \text{Imm}$（即 `jalr` 的目标地址），在下个上升沿锁存至 PC。
