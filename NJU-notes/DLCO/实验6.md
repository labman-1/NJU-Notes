# 6.1 控制器设计实验 
## 整体方案设计

### 一、 顶层设计模块图与译码数据流

控制器（Control Unit）是一个纯组合逻辑部件。其核心逻辑分为两级：
1.  **一级译码（指令类解析）**：通过对 7 位 `opcode` 进行数值比较，产生 9 个代表不同指令类型（Instruction Class）的单字位（1-bit）标志位。
2.  **二级译码（控制信号生成）**：结合这些 1 位指令类型标志位，以及功能码 `funct3`（3位）、`funct7[5]`（1位），通过最小项门电路阵列，瞬间收敛输出 9 路控制信号。

```text
                           控制器的输入端口
              +-----------------------------------------+
              | opcode[6:0]     funct3[2:0]   funct7[5] |
              +-----------------------------------------+
                    |                |           |
                    v                |           |
       +-------------------------+   |           |
       | 1级译码:指令类型生成阵列|   |           |
       |  (比较/与逻辑,如图6.1)  |   |           |
       +-------------------------+   |           |
                    |                |           |
     9路 1-bit 标志位 (U-lui, R-Arith, S-Instr等)  |
                    |                |           |
                    v                v           v
       +-------------------------------------------------+
       | 2级译码: 控制信号与功能码混合收敛逻辑门阵列      |
       |               (AND-OR-PLA 级联结构)             |
       +-------------------------------------------------+
          |        |        |        |        |        |        |        |        |
      ExtOp    RegWr  ALUASrc ALUBSrc  ALUctr  MemtoReg MemWr   MemOp   BandJ
         (3)      (1)      (1)      (2)      (4)      (1)      (1)      (3)      (3)
          |        |        |        |        |        |        |        |        |
          +--------+--------+--------+---输出----+--------+--------+--------+--------+
```

### 二、 子模块详细描述与技术实现

根据图 6.1 与图 6.2，整个控制器在物理上由三个核心逻辑子模块构成：

### 1. 核心一：一级指令类型译码器 (Instruction Type Decoder)
*   **功能描述**：对 7 位的输入操作码 `opcode` 进行硬件比对，判断当前指令属于 9 大指令类别中的哪一类，并输出高电平有效的 1 位控制线。
*   **实现方法**：
    在 Logisim 中，既可以通过“比较器 (Comparator)”组件来实现对特定 16 进制数的比对（如 `opcode == 0x37`），也可以直接通过**“分线器 + 多输入与/非门”**对这 7 位进行求与（例如对于 `lui` 的操作码 `0110111`，其译码公式为：$\text{U-lui} = \overline{op_6} \cdot op_5 \cdot op_4 \cdot \overline{op_3} \cdot op_2 \cdot op_1 \cdot op_1$）。
*   **指令类别与标志位映射**：
    *   `U-lui`     ($\text{opcode} = 37_{16}$ / $0110111_2$)：LUI 指令。
    *   `U-auipc`   ($\text{opcode} = 17_{16}$ / $0010111_2$)：AUIPC 指令。
    *   `I-Arith`   ($\text{opcode} = 13_{16}$ / $0010011_2$)：整数立即数运算指令。
    *   `R-Arith`   ($\text{opcode} = 33_{16}$ / $0110011_2$)：通用寄存器运算指令。
    *   `J-jal`     ($\text{opcode} = 6F_{16}$ / $1101111_2$)：JAL 无条件跳转。
    *   `J-jalr`    ($\text{opcode} = 67_{16}$ / $1100111_2$)：JALR 寄存器跳转。
    *   `B-Instr`   ($\text{opcode} = 63_{16}$ / $1100011_2$)：条件分支指令。
    *   `I-Load`    ($\text{opcode} = 03_{16}$ / $0000011_2$)：存储器加载指令。
    *   `S-Instr`   ($\text{opcode} = 23_{16}$ / $0100011_2$)：存储器写入指令。

### 2. 核心二：控制总线组合输出阵列 (Control Signals Output Array)
*   **功能描述**：结合 9 个分类标志位以及功能码 `funct3` / `funct7`，在**最小风险法（化简表达式，不利用无关项）**下，通过组合逻辑门输出 9 个基础控制引脚。

### 3. 核心三：下地址判定逻辑桥接 (NPC 控制器)
*   **功能描述**：输出控制跳转分支的 `BandJ`（3位）信号。该信号与 `lab5.5` 搭建的跳转控制模块对接，间接控制 `NPCASrc` 和 `NPCBSrc`。

### 三、 接口信号与引脚定义

### 1. 输入引脚
| 信号名称 | 方向 | 位宽 | 属性 | 物理功能与连接 |
| :--- | :---: | :---: | :---: | :--- |
| **opcode** | 输入 | 7 | 数据 | 来自指令拆解分线器的操作码字段 `IR[6:0]`。 |
| **funct3** | 输入 | 3 | 数据 | 来自指令拆解分线器的功能码字段 `IR[14:12]`。 |
| **funct7** | 输入 | 7 | 数据 | 来自指令拆解分线器的功能码字段 `IR[31:25]`。（实际译码仅用其第 5 位 `funct7[5]`）。 |

### 2. 输出引脚
| 信号名称 | 方向 | 位宽 | 属性 | 物理功能与输出逻辑（真值关系） |
| :--- | :---: | :---: | :---: | :--- |
| **ExtOp** | 输出 | 3 | 控制 | 立即数产生控制：`000` (I型), `001` (U型), `010` (S型), `011` (B型), `100` (J型)。 |
| **RegWr** | 输出 | 1 | 控制 | 寄存器写使能（LUI, AUIPC, I-Arith, R-Arith, JAL, JALR, Load 为 1；Branch, Store 为 0）。 |
| **ALUASrc** | 输出 | 1 | 控制 | ALU A输入端选择：`0` 代表寄存器数据，`1` 代表 PC（AUIPC, JAL, JALR 为 1）。 |
| **ALUBSrc** | 输出 | 2 | 控制 | ALU B输入端选择：`00` (寄存器), `01` (常数4, 用于JAL/JALR), `10` (立即数)。 |
| **ALUctr** | 输出 | 4 | 控制 | 指挥 ALU 运算类型。对于 R/I 型指令根据功能码动态生成，其他指令默认为加法 `0000`。 |
| **MemtoReg**| 输出 | 1 | 控制 | 写回源选择（仅 Load 指令为 1，写回内存读取数据）。 |
| **MemWr** | 输出 | 1 | 控制 | 数据内存写使能（仅 Store 类指令为 1）。 |
| **MemOp** | 输出 | 3 | 控制 | 访存字节/半字对齐控制（直接映射自 `funct3` 字段）。 |
| **BandJ** | 输出 | 3 | 控制 | 产生跳转控制器分支跳转信号（JAL=001, JALR=010, Branch=`1xx`）。 |
NPCASrc：下地址逻辑中加法器输入端 A 的选择信号，0：PC，1：R\[Rs1]。
NPCBSrc：下地址逻辑中加法器输入端 B 的选择信号，0：4，1：imm。

### 四、 控制器数据流向与动作协调分析

当 CPU 执行不同指令时，控制器将以如下逻辑通路进行高速、无时钟的静态选通：

### 1. 运算类指令（以 `addi` 为例）
*   输入 `opcode = 0010011` $\to$ 一级译码器激活标志位 `I-Arith = 1`。
*   控制通路输出：
    *   `ExtOp = 000` (扩展 I 型立即数)
    *   `RegWr = 1` (计算结果写回通用寄存器)
    *   `ALUASrc = 0` (ALU A 端选择通用寄存器 `rs1`)
    *   `ALUBSrc = 10` (ALU B 端选择立即数 `Imm`)
    *   `ALUctr` 结合 `funct3` 输出 `0000` (加法)
    *   `MemWr = 0` (内存不可写)

### 2. 条件跳转指令（以 `beq` 为例）
*   输入 `opcode = 1100011` $\to$ 激活标志位 `B-Instr = 1`。
*   控制通路输出：
    *   `ExtOp = 011` (B 型立即数扩展)
    *   `RegWr = 0` (不写回寄存器)
    *   `ALUBSrc = 00` (ALU 比较两个寄存器的数据)
    *   `ALUctr = 0010` (调用有符号减法作为比较手段)
    *   `BandJ = 100` (BEQ 类型分支，送往跳转控制器)

以下使用最小风险法计算各路控制信号的逻辑表达式。
最小风险法的核心是不利用任何无关项，仅对真值表中值为1的项进行化简。

我们要根据表 6.1，对 9 路控制信号分别写出最小项表达式。

首先，我们在第一级已经生成了 9 个指令类型的单字位（1-bit）标志位：
*   $U_{lui}$ (lui)
*   $U_{auipc}$ (auipc)
*   $I_{arith}$ (I型运算)
*   $R_{arith}$ (R型运算)
*   $J_{jal}$ (jal)
*   $J_{jalr}$ (jalr)
*   $B_{instr}$ (条件跳转)
*   $I_{load}$ (加载访存)
*   $S_{instr}$ (写入访存)

---

### 1. 寄存器写使能：`RegWr` 的表达式
*   **分析**：查看表 6.1，`RegWr` 在除 `B-Instr`（条件跳转）和 `S-Instr`（写入访存）外，其余所有指令下均明确为 **`1`**。
*   **最小项求和（最小风险法）**：我们只把所有对应 `RegWr = 1` 的指令类型进行“或”运算。
*   **表达式**：
    $$\text{RegWr} = U_{lui} + U_{auipc} + I_{arith} + R_{arith} + J_{jal} + J_{jalr} + I_{load}$$
*   *验证*：当输入是一条 Store 指令（`S-Instr`）时，`RegWr` 自动输出 `0`，不会发生误写寄存器的风险。

---

### 2. 内存写使能：`MemWr` 的表达式
*   **分析**：查看表 6.1，`MemWr` 只有在 `S-Instr`（即 `sb`, `sh`, `sw` 指令，`op = 0100011`）时才明确为 **`1`**。
*   **表达式**：
    $$\text{MemWr} = S_{instr}$$

---

### 3. 写回数据源：`MemtoReg` 的表达式
*   **分析**：`MemtoReg` 只有在 `I-load`（`lb`, `lh`, `lw`, `lbu`, `lhu` 指令）时才明确为 **`1`**，将内存读出的数据写回。其他指令要么是 ALU 结果（`0`），要么不写回（无关项 `×`）。
*   根据最小风险法：**无关项一律当 0 处理**。
*   **表达式**：
    $$\text{MemtoReg} = I_{load}$$

---

### 4. ALU A端输入源：`ALUASrc` 的表达式
*   **分析**：`ALUASrc` 在 `auipc`、`jal`、`jalr` 指令下需要将 `PC` 作为 A 端输入，即 `ALUASrc = 1`。其他为 0 或 ×。
*   **表达式**：
    $$\text{ALUASrc} = U_{auipc} + J_{jal} + J_{jalr}$$

---

### 5. ALU B端输入源：`ALUBSrc` 的表达式（2位宽）
因为 `ALUBSrc` 是两位宽，我们需要分别对高位 `ALUBSrc[1]` 和低位 `ALUBSrc[0]` 写表达式。

*   **低位 `ALUBSrc[0]`**：只有在 `jal` (`01`) 和 `jalr` (`01`) 时为 `1`。
    $$\text{ALUBSrc}[0] = J_{jal} + J_{jalr}$$
*   **高位 `ALUBSrc[1]`**：在所有立即数运算、Load、Store 指令（即需要使用扩展立即数 `Imm` 选通 `10` 通道）下为 `1`。
    $$\text{ALUBSrc}[1] = U_{lui} + U_{auipc} + I_{arith} + I_{load} + S_{instr}$$
*   **合流验证**：
    *   R型指令：`ALUBSrc[1] = 0`, `ALUBSrc[0] = 0` $\to$ 输出 `00`（选寄存器）。
    *   I型立即数：`ALUBSrc[1] = 1`, `ALUBSrc[0] = 0` $\to$ 输出 `10`（选立即数）。

---

### 6. 数据存储器读写长度：`MemOp` 的表达式（3位宽）
*   **分析**：查看表 6.1，`MemOp` 只有在 `I-load` 和 `S-instr` 时起作用，其取值完全与指令中的 **`funct3` 字段一致**。
*   由于最小风险法要求非定义指令下 `MemOp` 输出为 0：
*   **表达式**：
    $$\text{MemOp}[2:0] = (I_{load} + S_{instr}) \cdot \text{funct3}[2:0]$$
*   *实现*：将 `funct3` 信号通过一个与门阵列，由 `(I-load OR S-instr)` 信号作为选通。仅当执行访存指令时才允许 `funct3` 通过，否则输出全 0。

---

### 7. 跳转判定：`BandJ` 的表达式（3位宽）
需要分别计算 3 位输出：
*   `BandJ[0]`：在 `jal` (`001`)、`bne` (`101`)、`bge` (`111`)、`bgeu` (`111`) 时为 1。
    $$\text{BandJ}[0] = J_{jal} + B_{instr} \cdot (\text{funct3}[0])$$
*   `BandJ[1]`：在 `jalr` (`010`)、`blt/bltu` (`110`)、`bge/bgeu` (`111`) 时为 1。
    $$\text{BandJ}[1] = J_{jalr} + B_{instr} \cdot (\text{funct3}[2])$$
*   `BandJ[2]`：在所有条件分支指令 `B-Instr`（`1xx`）下高位为 1。
    $$\text{BandJ}[2] = B_{instr}$$

### 立即数扩展控制：`ExtOp[2:0]` 的表达式
根据表 6.1，`ExtOp` 有 3 位宽，对应关系如下：
*   `ExtOp = 000` (I-Type)：对应 `I-Arith`, `I-Load`, `jalr`。
*   `ExtOp = 001` (U-Type)：对应 `lui`, `auipc`。
*   `ExtOp = 010` (S-Type)：对应 `S-Instr` (sb, sh, sw)。
*   `ExtOp = 011` (B-Type)：对应 `B-Instr` (分支指令)。
*   `ExtOp = 100` (J-Type)：对应 `jal`。

我们利用最小风险法（不考虑无关项）分别求出 3 位的表达式：
*   **`ExtOp[0]`**：只有在 U-Type (`001`) 和 B-Type (`011`) 时为 1。
    $$\text{ExtOp}[0] = U_{lui} + U_{auipc} + B_{instr}$$
*   **`ExtOp[1]`**：只有在 S-Type (`010`) 和 B-Type (`011`) 时为 1。
    $$\text{ExtOp}[1] = S_{instr} + B_{instr}$$
*   **`ExtOp[2]`**：只有在 J-Type (`100`) 时为 1。
    $$\text{ExtOp}[2] = J_{jal}$$

---

### 算术逻辑控制：`ALUctr[3:0]` 的表达式
`ALUctr` 共有 4 位，控制 ALU 进行 16 种运算之一。它的生成需要**结合指令类型和功能码**：

根据表 6.1，我们对 4 位分别进行最小风险法推导：

*   **`ALUctr[0]`**（逻辑左/右移、与运算等）：在 `slli`/`sll` (`0001`)、`srli`/`srl` (`0101`)、`srai`/`sra` (`1101`)、`andi`/`and` (`0111`)，以及 `lui` (`1111`) 等指令下为 1。
    $$\text{ALUctr}[0] = U_{lui} + (I_{arith} + R_{arith}) \cdot \text{funct3}[0]$$
*   **`ALUctr[1]`**（小于比较、异或等）：在 `slti`/`slt` (`0010`)、`sltiu`/`sltu` (`0011`)、`xori`/`xor` (`0100` 不含)、`ori`/`or` (`0110`)、`andi`/`and` (`0111`)，以及 `beq/bne/blt/bge` 等所有 Branch 指令 (`0010`) 下为 1。
    $$\text{ALUctr}[1] = B_{instr} + (I_{arith} + R_{arith}) \cdot \text{funct3}[1] + U_{lui}$$
*   **`ALUctr[2]`**（右移、或运算等）：在 `srli`/`srl` (`0101`)、`srai`/`sra` (`1101`)、`ori`/`or` (`0110`)、`andi`/`and` (`0111`)，以及 `lui` (`1111`) 下为 1。
    $$\text{ALUctr}[2] = (I_{arith} + R_{arith}) \cdot \text{funct3}[2] + U_{lui}$$
*   **`ALUctr[3]`**（减法、算术右移等）：在 `sub` (`1000`)、`srai`/`sra` (`1101`)，以及 `lui` (`1111`) 下为 1。
    *   注意：算术右移是在逻辑右移的基础上，由于 `funct7[5] = 1` 触发的。
    $$\text{ALUctr}[3] = R_{arith} \cdot \overline{\text{funct3}[2]} \cdot \overline{\text{funct3}[1]} \cdot \overline{\text{funct3}[0]} \cdot \text{funct7}[5] + (I_{arith} + R_{arith}) \cdot \text{funct3}[2] \cdot \overline{\text{funct3}[1]} \cdot \text{funct3}[0] \cdot \text{funct7}[5] + U_{lui}$$
    *   *(注：这个表达式在物理上，就是用 `funct7[5]` 去跟对应的移位/减法条件做“与”运算。)*

6.1 组合逻辑控制器设计方案与信号收敛分析 
本控制器采用经典的硬线控制设计范式，整个译码架构由纯组合逻辑构成。为了在不拉长关键路径的前提下，实现对 RV32I 37条核心指令的物理控制，设计方案分为两个译码层级：
> 
1.   一级指令类译码 ：
	输入的操作码 `opcode` 并不直接参与最终控制线的计算，而是首先通过一组并行的**7位数值比较器（数字比较/与阵列）**。根据操作码的特征值，瞬间激活并输出 9 个代表不同指令族类的 1-bit 高电平标志位（如 `R-Arith`、`I-Load` 等）。这极大地收缩了控制器的输入维度，避免了后续逻辑产生“项数爆炸”。
> 
2.  **二级控制信号收敛**：
    根据**最小风险法（将未定义无关项一律强行置0）**，本设计对 9 路控制引脚的布线方程进行了严谨的门电路求和：
    *   对于通用的使能信号（如 `RegWr`, `MemWr`），直接通过一级标志位的**单级或门（OR）阵列**进行输出。
    *   对于与指令类型和操作码相关的多位宽信号（如 `ALUctr`, `MemOp`, `BandJ`），将 `funct3` 信号和 `funct7[5]` 作为使能网格的二重选通源，最终由低门延迟的组合逻辑收敛输出。
3.  **安全性评估**：
	由于全通路严格贯彻了最小风险法，若系统因外部干扰读取到非法非定义的操作码，控制器将安全输出 `RegWr = 0` 且 `MemWr = 0`，强行冻结 CPU 状态，有效地防止了程序跑飞对物理寄存器和数据存储造成误写入。
