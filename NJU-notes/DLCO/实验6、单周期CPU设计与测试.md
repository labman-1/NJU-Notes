# 6.1 控制器设计实验 
## 整体方案设计

### 一、 顶层设计模块图与译码数据流

控制器（Control Unit）是一个纯组合逻辑部件。其核心逻辑分为两级：
1.  **一级译码（指令类解析）**：通过对 7 位 `opcode` 进行数值比较，产生 9 个代表不同指令类型（Instruction Class）的单字位（1-bit）标志位。
2.  **二级译码（控制信号生成）**：结合这些 1 位指令类型标志位，以及功能码 `funct3`（3位）、`funct7[5]`（1位），通过最小项门电路阵列，瞬间收敛输出 9 路控制信号。

```text
                   控制器的输入端口
      +-----------------------------------------+
      | opcode[6:0]     funct3[2:0]   funct7[5] |
      +-----------------------------------------+
            |                |           |
            v                |           |
+-------------------------+  |           |
| 1级译码:指令类型生成阵列  |  |           |
|  (比较/与逻辑,如图6.1)   |  |           |
+-------------------------+  |           |
            |                |           |
9路 1-bit 标志位 (U-lui, R-Arith, S-Instr等)  |
            |                |           |
            v                v           v
+-------------------------------------------------+
| 2级译码: 控制信号与功能码混合收敛逻辑门阵列      |
|               (AND-OR-PLA 级联结构)             |
+-------------------------------------------------+
    |        |        |        |        |        |        |        |        |
ExtOp    RegWr  ALUASrc ALUBSrc  ALUctr  MemtoReg MemWr   MemOp   BandJ
    (3)      (1)      (1)      (2)      (4)      (1)      (1)      (3)      (3)
    |        |        |        |        |        |        |        |        |
    +--------+--------+--------+---输出----+--------+--------+--------+--------+
```

### 二、 子模块详细描述与技术实现

根据图 6.1 与图 6.2，整个控制器在物理上由三个核心逻辑子模块构成：

### 1. 核心一：一级指令类型译码器 
*   **功能描述**：对 7 位的输入操作码 `opcode` 进行硬件比对，判断当前指令属于 9 大指令类别中的哪一类，并输出高电平有效的 1 位控制线。
*   **实现方法**：
    在 Logisim 中，既可以通过“比较器 (Comparator)”组件来实现对特定 16 进制数的比对（如 `opcode == 0x37`），也可以直接通过**“分线器 + 多输入与/非门”**对这 7 位进行求与（例如对于 `lui` 的操作码 `0110111`，其译码公式为：$\text{U-lui} = \overline{op_6} \cdot op_5 \cdot op_4 \cdot \overline{op_3} \cdot op_2 \cdot op_1 \cdot op_1$）。
*   **指令类别与标志位映射**：
    *   `U-lui`     ($\text{opcode} = 37_{16}$ / $0110111_2$)：LUI 指令。
    *   `U-auipc`   ($\text{opcode} = 17_{16}$ / $0010111_2$)：AUIPC 指令。
    *   `I-Arith`   ($\text{opcode} = 13_{16}$ / $0010011_2$)：整数立即数运算指令。
    *   `R-Arith`   ($\text{opcode} = 33_{16}$ / $0110011_2$)：通用寄存器运算指令。
    *   `J-jal`     ($\text{opcode} = 6F_{16}$ / $1101111_2$)：JAL 无条件跳转。
    *   `J-jalr`    ($\text{opcode} = 67_{16}$ / $1100111_2$)：JALR 寄存器跳转。
    *   `B-Instr`   ($\text{opcode} = 63_{16}$ / $1100011_2$)：条件分支指令。
    *   `I-Load`    ($\text{opcode} = 03_{16}$ / $0000011_2$)：存储器加载指令。
    *   `S-Instr`   ($\text{opcode} = 23_{16}$ / $0100011_2$)：存储器写入指令。

### 2. 核心二：控制总线组合输出阵列
*   **功能描述**：结合 9 个分类标志位以及功能码 `funct3` / `funct7`，在**最小风险法（化简表达式，不利用无关项）** 下，通过组合逻辑门输出 9 个基础控制引脚。

### 3. 核心三：下地址判定逻辑桥接 (NPC 控制器)
*   **功能描述**：输出控制跳转分支的 `BandJ`（3位）信号。该信号与 `lab5.5` 搭建的跳转控制模块对接，间接控制 `NPCASrc` 和 `NPCBSrc`。

### 三、 接口信号与引脚定义

### 1. 输入引脚
| 信号名称 | 方向 | 位宽 | 属性 | 物理功能与连接 |
| :--- | :---: | :---: | :---: | :--- |
| **opcode** | 输入 | 7 | 数据 | 来自指令拆解分线器的操作码字段 `IR[6:0]`。 |
| **funct3** | 输入 | 3 | 数据 | 来自指令拆解分线器的功能码字段 `IR[14:12]`。 |
| **funct7** | 输入 | 7 | 数据 | 来自指令拆解分线器的功能码字段 `IR[31:25]`。（实际译码仅用其第 5 位 `funct7[5]`）。 |

### 2. 输出引脚
| 信号名称 | 方向 | 位宽 | 属性 | 物理功能与输出逻辑（真值关系） |
| :--- | :---: | :---: | :---: | :--- |
| **ExtOp** | 输出 | 3 | 控制 | 立即数产生控制：`000` (I型), `001` (U型), `010` (S型), `011` (B型), `100` (J型)。 |
| **RegWr** | 输出 | 1 | 控制 | 寄存器写使能（LUI, AUIPC, I-Arith, R-Arith, JAL, JALR, Load 为 1；Branch, Store 为 0）。 |
| **ALUASrc** | 输出 | 1 | 控制 | ALU A输入端选择：`0` 代表寄存器数据，`1` 代表 PC（AUIPC, JAL, JALR 为 1）。 |
| **ALUBSrc** | 输出 | 2 | 控制 | ALU B输入端选择：`00` (寄存器), `01` (常数4, 用于JAL/JALR), `10` (立即数)。 |
| **ALUctr** | 输出 | 4 | 控制 | 指挥 ALU 运算类型。对于 R/I 型指令根据功能码动态生成，其他指令默认为加法 `0000`。 |
| **MemtoReg**| 输出 | 1 | 控制 | 写回源选择（仅 Load 指令为 1，写回内存读取数据）。 |
| **MemWr** | 输出 | 1 | 控制 | 数据内存写使能（仅 Store 类指令为 1）。 |
| **MemOp** | 输出 | 3 | 控制 | 访存字节/半字对齐控制（直接映射自 `funct3` 字段）。 |
| **BandJ** | 输出 | 3 | 控制 | 产生跳转控制器分支跳转信号（JAL=001, JALR=010, Branch=`1xx`）。 |
NPCASrc：下地址逻辑中加法器输入端 A 的选择信号，0：PC，1：R\[Rs1]。
NPCBSrc：下地址逻辑中加法器输入端 B 的选择信号，0：4，1：imm。

### 四、 控制器数据流向与动作协调分析

当 CPU 执行不同指令时，控制器将以如下逻辑通路进行高速、无时钟的静态选通：

### 1. 运算类指令（以 `addi` 为例）
*   输入 `opcode = 0010011` $\to$ 一级译码器激活标志位 `I-Arith = 1`。
*   控制通路输出：
    *   `ExtOp = 000` (扩展 I 型立即数)
    *   `RegWr = 1` (计算结果写回通用寄存器)
    *   `ALUASrc = 0` (ALU A 端选择通用寄存器 `rs1`)
    *   `ALUBSrc = 10` (ALU B 端选择立即数 `Imm`)
    *   `ALUctr` 结合 `funct3` 输出 `0000` (加法)
    *   `MemWr = 0` (内存不可写)

### 2. 条件跳转指令（以 `beq` 为例）
*   输入 `opcode = 1100011` $\to$ 激活标志位 `B-Instr = 1`。
*   控制通路输出：
    *   `ExtOp = 011` (B 型立即数扩展)
    *   `RegWr = 0` (不写回寄存器)
    *   `ALUBSrc = 00` (ALU 比较两个寄存器的数据)
    *   `ALUctr = 0010` (调用有符号减法作为比较手段)
    *   `BandJ = 100` (BEQ 类型分支，送往跳转控制器)

以下使用最小风险法计算各路控制信号的逻辑表达式。
最小风险法的核心是不利用任何无关项，仅对真值表中值为1的项进行化简。

我们要根据表 6.1，对 9 路控制信号分别写出最小项表达式。

首先，我们在第一级已经生成了 9 个指令类型的单字位（1-bit）标志位：
*   $U_{lui}$ (lui)
*   $U_{auipc}$ (auipc)
*   $I_{arith}$ (I型运算)
*   $R_{arith}$ (R型运算)
*   $J_{jal}$ (jal)
*   $J_{jalr}$ (jalr)
*   $B_{instr}$ (条件跳转)
*   $I_{load}$ (加载访存)
*   $S_{instr}$ (写入访存)

---

### 1. 寄存器写使能：`RegWr` 的表达式
*   **分析**：查看表 6.1，`RegWr` 在除 `B-Instr`（条件跳转）和 `S-Instr`（写入访存）外，其余所有指令下均明确为 **`1`**。
*   **最小项求和（最小风险法）**：我们只把所有对应 `RegWr = 1` 的指令类型进行“或”运算。
*   **表达式**：
    $$\text{RegWr} = U_{lui} + U_{auipc} + I_{arith} + R_{arith} + J_{jal} + J_{jalr} + I_{load}$$
*   *验证*：当输入是一条 Store 指令（`S-Instr`）时，`RegWr` 自动输出 `0`，不会发生误写寄存器的风险。

---

### 2. 内存写使能：`MemWr` 的表达式
*   **分析**：查看表 6.1，`MemWr` 只有在 `S-Instr`（即 `sb`, `sh`, `sw` 指令，`op = 0100011`）时才明确为 **`1`**。
*   **表达式**：
    $$\text{MemWr} = S_{instr}$$

---

### 3. 写回数据源：`MemtoReg` 的表达式
*   **分析**：`MemtoReg` 只有在 `I-load`（`lb`, `lh`, `lw`, `lbu`, `lhu` 指令）时才明确为 **`1`**，将内存读出的数据写回。其他指令要么是 ALU 结果（`0`），要么不写回（无关项 `×`）。
*   根据最小风险法：**无关项一律当 0 处理**。
*   **表达式**：
    $$\text{MemtoReg} = I_{load}$$

---

### 4. ALU A端输入源：`ALUASrc` 的表达式
*   **分析**：`ALUASrc` 在 `auipc`、`jal`、`jalr` 指令下需要将 `PC` 作为 A 端输入，即 `ALUASrc = 1`。其他为 0 或 ×。
*   **表达式**：
    $$\text{ALUASrc} = U_{auipc} + J_{jal} + J_{jalr}$$

---

### 5. ALU B端输入源：`ALUBSrc` 的表达式（2位宽）
因为 `ALUBSrc` 是两位宽，我们需要分别对高位 `ALUBSrc[1]` 和低位 `ALUBSrc[0]` 写表达式。

*   **低位 `ALUBSrc[0]`**：只有在 `jal` (`01`) 和 `jalr` (`01`) 时为 `1`。
    $$\text{ALUBSrc}[0] = J_{jal} + J_{jalr}$$
*   **高位 `ALUBSrc[1]`**：在所有立即数运算、Load、Store 指令（即需要使用扩展立即数 `Imm` 选通 `10` 通道）下为 `1`。
    $$\text{ALUBSrc}[1] = U_{lui} + U_{auipc} + I_{arith} + I_{load} + S_{instr}$$
*   **合流验证**：
    *   R型指令：`ALUBSrc[1] = 0`, `ALUBSrc[0] = 0` $\to$ 输出 `00`（选寄存器）。
    *   I型立即数：`ALUBSrc[1] = 1`, `ALUBSrc[0] = 0` $\to$ 输出 `10`（选立即数）。

---

### 6. 数据存储器读写长度：`MemOp` 的表达式（3位宽）
*   **分析**：查看表 6.1，`MemOp` 只有在 `I-load` 和 `S-instr` 时起作用，其取值完全与指令中的 **`funct3` 字段一致**。
*   由于最小风险法要求非定义指令下 `MemOp` 输出为 0：
*   **表达式**：
    $$\text{MemOp}[2:0] = (I_{load} + S_{instr}) \cdot \text{funct3}[2:0]$$
*   *实现*：将 `funct3` 信号通过一个与门阵列，由 `(I-load OR S-instr)` 信号作为选通。仅当执行访存指令时才允许 `funct3` 通过，否则输出全 0。

---

### 7. 跳转判定：`BandJ` 的表达式（3位宽）
需要分别计算 3 位输出：
*   `BandJ[0]`：在 `jal` (`001`)、`bne` (`101`)、`bge` (`111`)、`bgeu` (`111`) 时为 1。
    $$\text{BandJ}[0] = J_{jal} + B_{instr} \cdot (\text{funct3}[0])$$
*   `BandJ[1]`：在 `jalr` (`010`)、`blt/bltu` (`110`)、`bge/bgeu` (`111`) 时为 1。
    $$\text{BandJ}[1] = J_{jalr} + B_{instr} \cdot (\text{funct3}[2])$$
*   `BandJ[2]`：在所有条件分支指令 `B-Instr`（`1xx`）下高位为 1。
    $$\text{BandJ}[2] = B_{instr}$$

### 立即数扩展控制：`ExtOp[2:0]` 的表达式
根据表 6.1，`ExtOp` 有 3 位宽，对应关系如下：
*   `ExtOp = 000` (I-Type)：对应 `I-Arith`, `I-Load`, `jalr`。
*   `ExtOp = 001` (U-Type)：对应 `lui`, `auipc`。
*   `ExtOp = 010` (S-Type)：对应 `S-Instr` (sb, sh, sw)。
*   `ExtOp = 011` (B-Type)：对应 `B-Instr` (分支指令)。
*   `ExtOp = 100` (J-Type)：对应 `jal`。

我们利用最小风险法（不考虑无关项）分别求出 3 位的表达式：
*   **`ExtOp[0]`**：只有在 U-Type (`001`) 和 B-Type (`011`) 时为 1。
    $$\text{ExtOp}[0] = U_{lui} + U_{auipc} + B_{instr}$$
*   **`ExtOp[1]`**：只有在 S-Type (`010`) 和 B-Type (`011`) 时为 1。
    $$\text{ExtOp}[1] = S_{instr} + B_{instr}$$
*   **`ExtOp[2]`**：只有在 J-Type (`100`) 时为 1。
    $$\text{ExtOp}[2] = J_{jal}$$

---

### 算术逻辑控制：`ALUctr[3:0]` 的表达式
`ALUctr` 共有 4 位，控制 ALU 进行 16 种运算之一。它的生成需要**结合指令类型和功能码**：

根据表 6.1，我们对 4 位分别进行最小风险法推导：

*   **`ALUctr[0]`**（逻辑左/右移、与运算等）：在 `slli`/`sll` (`0001`)、`srli`/`srl` (`0101`)、`srai`/`sra` (`1101`)、`andi`/`and` (`0111`)，以及 `lui` (`1111`) 等指令下为 1。
    $$\text{ALUctr}[0] = U_{lui} + (I_{arith} + R_{arith}) \cdot \text{funct3}[0]+B_{instr}\cdot func3[1]$$
*   **`ALUctr[1]`**（小于比较、异或等）：在 `slti`/`slt` (`0010`)、`sltiu`/`sltu` (`0011`)、`xori`/`xor` (`0100` 不含)、`ori`/`or` (`0110`)、`andi`/`and` (`0111`)，以及 `beq/bne/blt/bge` 等所有 Branch 指令 (`0010`) 下为 1。
    $$\text{ALUctr}[1] = B_{instr} + (I_{arith} + R_{arith}) \cdot \text{funct3}[1] + U_{lui}$$
*   **`ALUctr[2]`**（右移、或运算等）：在 `srli`/`srl` (`0101`)、`srai`/`sra` (`1101`)、`ori`/`or` (`0110`)、`andi`/`and` (`0111`)，以及 `lui` (`1111`) 下为 1。
    $$\text{ALUctr}[2] = (I_{arith} + R_{arith}) \cdot \text{funct3}[2] + U_{lui}$$
*   **`ALUctr[3]`**（减法、算术右移等）：在 `sub` (`1000`)、`srai`/`sra` (`1101`)，以及 `lui` (`1111`) 下为 1。
    *   注意：算术右移是在逻辑右移的基础上，由于 `funct7[5] = 1` 触发的。
    $$\text{ALUctr}[3] = R_{arith} \cdot \overline{\text{funct3}[2]} \cdot \overline{\text{funct3}[1]} \cdot \overline{\text{funct3}[0]} \cdot \text{funct7}[5] + (I_{arith} + R_{arith}) \cdot \text{funct3}[2] \cdot \overline{\text{funct3}[1]} \cdot \text{funct3}[0] \cdot \text{funct7}[5] + U_{lui}$$
    *   *(注：这个表达式在物理上，就是用 `funct7[5]` 去跟对应的移位/减法条件做“与”运算。)*

6.1 组合逻辑控制器设计方案与信号收敛分析 
本控制器采用经典的硬线控制设计范式，整个译码架构由纯组合逻辑构成。为了在不拉长关键路径的前提下，实现对 RV32I 37条核心指令的物理控制，设计方案分为两个译码层级：
> 
1.   一级指令类译码 ：
	输入的操作码 `opcode` 并不直接参与最终控制线的计算，而是首先通过一组并行的**7位数值比较器（数字比较/与阵列）**。根据操作码的特征值，瞬间激活并输出 9 个代表不同指令族类的 1-bit 高电平标志位（如 `R-Arith`、`I-Load` 等）。这极大地收缩了控制器的输入维度，避免了后续逻辑产生“项数爆炸”。
> 
2.  **二级控制信号收敛**：
    根据**最小风险法（将未定义无关项一律强行置0）**，本设计对 9 路控制引脚的布线方程进行了严谨的门电路求和：
    *   对于通用的使能信号（如 `RegWr`, `MemWr`），直接通过一级标志位的**单级或门（OR）阵列**进行输出。
    *   对于与指令类型和操作码相关的多位宽信号（如 `ALUctr`, `MemOp`, `BandJ`），将 `funct3` 信号和 `funct7[5]` 作为使能网格的二重选通源，最终由低门延迟的组合逻辑收敛输出。
3.  **安全性评估**：
	由于全通路严格贯彻了最小风险法，若系统因外部干扰读取到非法非定义的操作码，控制器将安全输出 `RegWr = 0` 且 `MemWr = 0`，强行冻结 CPU 状态，有效地防止了程序跑飞对物理寄存器和数据存储造成误写入。

### 二、 子模块详细描述

#### 1. 升级版取指部件
*   **功能描述**：用于保存和更新当前执行指令的物理地址。
*   **硬件修改细节**：
    *   在 `IFU` 内部，将原本恒接常量 `1` 的 `PC 寄存器` 的 **写使能端（`en`）**，连接到新增的输入引脚 **`~Halt`**（中止信号经过一个非门）。
    *   *时序控制机制*：当系统正常运行时，`Halt = 0`，`en = 1`，PC 在每个时钟上升沿随 `NextPC` 更新。当执行到 `ecall` 指令时，控制器使 `Halt` 变为了 `1`，`en` 变为 `0`。在下一个时钟上升沿到来时，PC 寄存器拒绝采样新数据，CPU 硬件在物理上实现“瞬间休眠”。

#### 2. 升级版控制部件 (Controller - Upgraded)
*   **功能描述**：接收 `opcode[6:0]`，产生整机动作信号。
*   **硬件修改细节**：
    *   增加一个 1 位输出引脚 **`Halt`**。
    *   **译码逻辑公式**：
        $$\text{Halt} = (\text{opcode}[6:0] == 1110011_2) = (\text{opcode} == 0x73)$$
        这对应了 RISC-V 规范中的环境调用指令 **`ecall`**。
    *   *最小风险实现*：利用一个 7 输入与门，对输入 `opcode` 的位（`1110011`）进行特征检测，直接连到 `Halt` 端口。

#### 3. 数据存储器部件 (DataRAM - 256KB)
*   **功能描述**：字节编址的 RAM 阵列，提供灵活的字节（sb/lb）、半字（sh/lh）和字（sw/lw）访存。
*   **控制对齐**：其控制端 `MemOp` 连接控制器的输出。当全局复位信号 `RESET` 有效时，其清零端 `clr` 被置为 1，确保内存初始化。

#### 4. 运行周期计数器部件 (Cycle Counter)
*   **功能描述**：用于量化和评估程序执行效率的辅助测试部件。
*   **硬件连线实现**：
    *   放一个 Logisim 的 **16位/32位 计数器 (Counter)**。
    *   其时钟端 `CLK` 连接系统主时钟 `clk`。
    *   其清零端 `clr`（或 `RST`）连接全局复位信号 `RESET`。
    *   其使能控制端 `en` 连接 **`~Halt`**（当系统停机时，计数器同步停止计数）。

# 6.3 累加和程序测试实验
## 整体方案设计

## 1. 顶层设计方案概述

本实验的目标是在已实现的单周期 RV32I CPU 基础上，加载并运行自研的累加和（$S = 1 + 2 + \dots + n$）汇编程序，验证单周期 CPU 的控制通路与数据通路在真实复杂程序环境下的正确性与协同工作能力。

单周期 CPU 的核心思想是**所有指令的执行（取指、译码、执行、访存、写回）均在一个时钟周期内完成**。在实验 6.3 中，顶层设计将指令存储器（ROM）与数据存储器（DataRAM）独立部署。程序在运行前，累加和算法的机器码已被预加载至 ROM 中；参数 $n$ 被预写入 DataRAM 的 `0x0` 地址处。启动自动仿真后，CPU 按照时钟信号驱动逐步执行程序，计算结果 $S$ 最终会被写入 DataRAM 的 `0x4` 地址处。当程序执行到 `ecall`（停机指令）时，控制器拉高 `Halt` 信号，从而锁定 PC 寄存器及周期计数器，完成程序的安全退出与状态保持。

---

## 2. 顶层设计模块图与数据/控制通路

单周期 CPU 顶层电路由 **取指部件 (IFU)**、**指令存储器 (ROM)**、**译码与寄存器堆部件 (IDU)**、**算术逻辑单元 (ALU)**、**数据存储器 (DataRAM)**、**控制器 (Controller)** 以及**执行周期计数器**等关键子模块构成。

### 2.1 顶层电路逻辑拓扑图（结构简述）
根据 Logisim 的顶层引脚布局（图 6.5），顶层模块之间的物理连接关系及信号流动方向如下：

```
                +-------------------+
                |       控制器       | <-----------------------------+
                |    (Controller)   |                                | OP/funct3/funct7
                +-------------------+                                |
                  |  |  |  |  |  |                                   |
    +-------------+  |  |  |  |  +------------+                      |
    | 控制信号       |  |  |  |               | 控制信号              |
    v                v  v  v  v               v                      |
+-------+   PC     +------------+   Instr   +------------+  Result0  |
|  IFU  | -------> |  64KB ROM  | --------> |    IDU     | ----------+
+-------+          +------------+           | (RegFile)  |
    ^                                       +------------+
    | NPCASrc/NPCBSrc/Imm/BusA                    |  | DataA/DataB
    +---------------------------------------------+  v
                                                +------------+
                                                |    ALU     |
                                                +------------+
                                                     | S (Addr)
                                                     v
                                                +------------+  Dout   +-------+
                                                |  DataRAM   | ------> |  MUX  | ----> BusW (写回)
                                                +------------+         +-------+
```

### 2.2 两大通路划分
*   **控制通路 (Control Path)**：
    指令寄存器输出的指令 `IR` 被送入控制器，控制器通过对操作码 `OP`（`IR[6:0]`）、功能码 `funct3`（`IR[14:12]`）以及 `funct7[5]`（`IR[30]`）进行组合逻辑译码，产生控制全局的信号：`ExtOp`、`RegWr`、`ALUASrc`、`ALUBSrc`、`ALUctr`、`MemtoReg`、`MemWr`、`MemOp`、`BandJ` 和 `Halt`。
*   **数据通路 (Data Path)**：
    *   **取指阶段**：IFU 产生当前 `PC`，地址送入 ROM 读取 32 位指令 `IR`。
    *   **译码/读寄存器阶段**：IDU 根据 `IR` 的寄存器源地址读取 `DataA`（`R[rs1]`）和 `DataB`（`R[rs2]`），并通过立即数生成器（ImmGen）根据 `ExtOp` 产生扩展后的 32 位立即数 `Imm`。
    *   **执行阶段**：多路选择器根据 `ALUASrc` 和 `ALUBSrc` 选择运算操作数送入 ALU，ALU 在 `ALUctr` 的控制下完成计算，输出运算结果 `Result0`（或 `S`）及分支标志 `Zero`。
    *   **访存阶段**：若为访存指令，ALU 计算出的地址送入 DataRAM。写指令时将寄存器读出的 `Din`（即 `R[rs2]`）写入对应存储单元；读指令时从 DataRAM 读出 32 位数据 `Dout`。
    *   **写回阶段**：多路选择器根据 `MemtoReg` 选择将 ALU 结果或 DataRAM 读取的数据写回目的寄存器 `R[rd]`，写使能由 `RegWr` 控制。

## 3. 子模块详细描述与引脚定义

### 3.1 取指部件 (IFU)
*   **详细描述**：
    负责计算并输出下一条指令的地址。包含一个 32 位 PC 寄存器、一个 32 位加法器以及多路选择器。
    *   加法器输入端 A 由 `NPCASrc` 控制，在 `0` 时选择当前 `PC`，在 `1` 时选择寄存器堆输出的 `BusA`（用于 `jalr` 指令）。
    *   加法器输入端 B 由 `NPCBSrc` 控制，在 `0` 时选择固定常数 `4`，在 `1` 时选择跳转偏移立即数 `Imm`（用于分支/跳转指令）。
    *   PC 寄存器具有写使能端 `en`，该端与外部 `~halt` 信号相连。当检测到 `Halt` 信号有效（`halt = 1`）时，`en` 变为 `0`，PC 锁存当前值，停止更新。
    *   具有异步复位信号 `RESET`，当 `RESET = 1` 时，PC 被强置为程序初始地址 `Init Addr`。
*   **引脚定义**：
    *   `Init Addr` \[32位，输入]：程序初始加载地址。
    *   `Imm` \[32位，输入]：译码得到的 32 位分支/跳转偏移量。
    *   `BusA` \[32位，输入]：来自寄存器堆的 `R[rs1]`，用于计算 `jalr` 的目标地址。
    *   `CLK` \[1位，输入]：系统时钟信号（上升沿触发更新 PC）。
    *   `Reset` \[1位，输入]：异步复位信号。
    *   `NPCASrc` \[1位，输入]：下地址加法器 A 输入多路选择信号。
    *   `NPCBSrc` \[1位，输入]：下地址加法器 B 输入多路选择信号。
    *   `halt` \[1位，输入]：停机控制信号，高电平有效。
    *   `PC` \[32位，输出]：当前指令地址，送往指令存储器 ROM。

### 3.2 译码与寄存器堆部件 (IDU / RegFile)
*   **详细描述**：
    该部件集成了 32 个 32 位通用寄存器（`x0` 恒为 0）和立即数扩展器。
    *   根据 `IR` 中的 `rs1`（`IR[19:15]`）、`rs2`（`IR[24:20]`）和 `rd`（`IR[11:7]`）地址，实现双端口读和一个端口写。
    *   立即数扩展模块依据控制器输出的 `ExtOp` 编码，将 `IR` 中的不同字段拼接并符号扩展（或零扩展）为标准 32 位立即数 `Imm`。
    *   在写回阶段，若 `RegWr = 1`，则在时钟上升沿将 `BusW` 写入目的寄存器 `rd`。
*   **引脚定义**：
    *   `IR` \[32位，输入]：当前执行的指令字。
    *   `PC` \[32位，输入]：当前指令的 PC 值，用于 `auipc` 或 `jal`/`jalr` 的写回计算。
    *   `BusW` \[32位，输入]：准备写入目的寄存器的数据。
    *   `ExtOp` \[3位，输入]：立即数生成模式控制信号。
    *   `RegWr` \[1位，输入]：寄存器堆写使能信号。
    *   `CLK` \[1位，输入]：时钟信号。
    *   `ALUASrc` \[1位，输入]：ALU 输入端 A 的源选择控制。
    *   `ALUBSrc` \[2位，输入]：ALU 输入端 B 的源选择控制。
    *   `Imm` \[32位，输出]：扩展后的 32 位立即数。
    *   `BusA` \[32位，输出]：寄存器 `R[rs1]` 的原始输出，直接连至 IFU 用于 `jalr` 目标地址计算。
    *   `DataA` \[32位，输出]：送入 ALU 输入 A 端的数据（根据 `ALUASrc` 可能是 `R[rs1]` 或 `PC`）。
    *   `DataB` \[32位，输出]：连至访存写数据端 `Din` 的数据（即 `R[rs2]`）。
    *   `BusB` \[32位，输出]：送入 ALU 输入 B 端的数据（根据 `ALUBSrc` 可能是 `R[rs2]`、`4` 或 `Imm`）。

### 3.3 控制器 (Controller)
*   **详细描述**：
    控制器的输入为指令操作码和功能码。由于 RV32I 指令类型可通过其操作码 `OP` 分类，控制器内部首先对 7 位操作码进行全译码，产生各指令类型的使能信号：`U_lui`、`U_auipc`、`I_arith`、`R_arith`、`J_jal`、`J_jalr`、`B_instr`、`I_load`、`S_instr`。
    *   针对本实验 6.3 的需求，特别设计了**非标准异常/停机检测逻辑**：当指令操作码 `OP` 为 `1110011`（0x73，对应汇编中的 `ecall`）时，控制器将输出信号 `Halt` 置为高电平 `1`，用以中止处理器运行。
    *   其余控制信号的逻辑产生严格按照表 6.1 与最小风险化简法实现。
*   **引脚定义**：
    *   `op` \[7位，输入]：指令操作码 `IR[6:0]`。
    *   `funct3` \[3位，输入]：指令功能码 `IR[14:12]`。
    *   `funct7[5]` \[1位，输入]：指令功能码第 30 位 `IR[30]`。
    *   `ExtOp` \[3位，输出]：立即数扩展控制。
    *   `RegWr` \[1位，输出]：寄存器堆写控制。
    *   `ALUASrc` \[1位，输出]：ALU 操作数 A 选择。
    *   `ALUBSrc` \[2位，输出]：ALU 操作数 B 选择。
    *   `ALUctr` \[4位，输出]：ALU 算术逻辑运算功能码。
    *   `MemtoReg` \[1位，输出]：寄存器堆写回数据源选择。
    *   `MemWr` \[1位，输出]：数据存储器写使能。
    *   `MemOp` \[3位，输出]：访存格式控制（字节/半字/字，符号/无符号扩展）。
    *   `BandJ` \[3位，输出]：分支跳转逻辑控制。
    *   `Halt` \[1位，输出]：停机中止信号。

### 3.4 算术逻辑单元 (ALU)
*   **详细描述**：
    纯组合逻辑运算部件，根据 `ALUctr` 控制信号对输入端 `A` 和 `B` 执行指定的 32 位运算。在分支指令执行中，ALU 通过计算结果 `Result0` 是否为 `0` 或比较符号位来辅助判断跳转条件。
*   **引脚定义**：
    *   `A` \[32位，输入]：源操作数 A。
    *   `B` \[32位，输入]：源操作数 B。
    *   `ALUctr` \[4位，输入]：运算控制码。
    *   `S` \[32位，输出]：运算结果，兼作 DataRAM 读写地址。
    *   `Zero` \[1位，输出]：零标志位，当 `S = 0` 时输出为 1。

### 3.5 数据存储器 (DataRAM)
*   **详细描述**：
    本模块在内部采用 4 片独立的 8 位宽（单字节）RAM（RAM0~RAM3）拼装实现，采用小端存储模式（Little-Endian）。
    *   通过 `MemOp` 控制信号和地址的低 2 位，内部片选译码电路可以精准控制哪几片 RAM 处于使能状态。
    *   支持字节（Byte，8位）、半字（Half-word，16位）以及字（Word，32位）的读写，并根据 `MemOp` 要求对读出的单字节或半字进行有符号或无符号位扩展。
    *   在累加和测试程序中，RAM 的 `0x0` 地址处初始化加载参数 `n`，计算完成后，累加和 `S` 会通过 `sw` 指令写入 `0x4` 地址（占用 RAM0~RAM3 的 `0x4` 地址字节空间）。
*   **引脚定义**：
    *   `Addr` \[32位，输入]：读写内存的目标地址。
    *   `Din` \[32位，输入]：要写入存储器的数据。
    *   `WE`\[1位，输入]：写使能（来自 `MemWr`）。
    *   `CLK` \[1位，输入]：时钟信号。
    *   `Clr` \[1位，输入]：清零复位信号。
    *   `MemOp` \[3位，输入]：读写位宽及扩展控制码。
    *   `Dout` \[32位，输出]：读出的 32 位扩展数据。

### 3.6 周期计数器 (Cycles Counter)
*   **详细描述**：
    专为统计程序执行效率设计的 16 位/32 位计数器。
    *   当异步复位信号 `RESET` 有效时，计数器清零。
    *   在正常运行阶段，每个时钟上升沿计数器自增 1。
    *   当 `Halt` 停机信号有效时，计数器的时钟或计数使能端被封锁，计数器停止工作，从而精确锁定累加和程序运行的总周期数。
*   **引脚定义**：
    *   `clk` [1位，输入]：系统时钟信号。
    *   `RESET` [1位，输入]：复位清零端。
    *   `Halt` [1位，输入]：计数暂停控制信号。
    *   `cycles` [32位，输出]：程序运行占用的总周期数。

## 4. 数据与控制信号传输通道定义

为说明 6.3 累加和程序如何在上述模块间进行数据流动，特定义程序核心阶段的传输通路：

1.  **参数读取通路 (`lw a1, 0(x0)`)**：
    *   `PC` $\rightarrow$ ROM 地址输入 $\rightarrow$ 读出 `IR` (`0x00002583`)。
    *   控制器接收到该操作码，识别为 `I_load`，使能控制信号：`ALUASrc = 0`（选择 `R[rs1]` 即 `x0`）、`ALUBSrc = 10`（选择立即数 `0`）、`ALUctr = 0000`（执行加法）、`MemtoReg = 1`（写回源来自内存）、`RegWr = 1`。
    *   ALU 计算出地址 `0`，送往 DataRAM 的 `Addr` 端，在 `MemOp = 000`（按字读取）控制下，DataRAM 从小端内存的 `0x0` 地址处读取 32 位参数 `n`。
    *   数据经过 `MemtoReg` 多路选择器，由 `BusW` 写入目的寄存器 `a1`（`x11`）。
2.  **循环计算通路 (`add a3, a3, a2`)**：
    *   `IR` (`0x00e686b3`) 经过译码：源寄存器为 `a3`（`x13`）和 `a2`（`x12`），目的寄存器为 `a3`（`x13`）。
    *   控制器产生 `ALUASrc = 0`，`ALUBSrc = 00`，`ALUctr = 0000`（加法），`RegWr = 1`，`MemtoReg = 0`。
    *   寄存器堆送出 `R[a3]`（当前累加和）与 `R[a2]`（当前循环变量 `i`），ALU 完成加法计算得到新累加值，结果通过数据选择器直接送回 `BusW` 写回寄存器 `a3`。
3.  **结果保存通路 (`sw a3, 4(x0)`)**：
    *   `IR` (`0x00d02223`) 译码：源寄存器为 `a3`（计算结果）和 `x0`，偏移量为 `4`。
    *   控制器输出 `ALUASrc = 0`，`ALUBSrc = 10`（选择立即数 `4`），`ALUctr = 0000`（地址计算），`MemWr = 1`，`MemOp = 000`（按字写入），`RegWr = 0`。
    *   ALU 输出目标地址 `4`，送入 DataRAM 的 `Addr` 端。同时，寄存器堆将 `R[a3]` 的值通过 `Din` 送往 DataRAM 的数据输入端。在时钟上升沿，累加值被成功存储在 DataRAM 的 `0x4` 物理空间内。
4.  **程序中止通路 (`ecall`)**：
    *   当 `PC` 指向停机指令时，ROM 输出 `IR` 值为 `0x00000073`。
    *   控制器全译码出 `Halt` 指令标志，产生 `Halt = 1`。
    *   该信号直接输出到顶层，在时钟通路中将 IFU 内 PC 寄存器的写使能端（`en`）拉低，并在周期计数器端封锁计数使能。此时整个 CPU 状态“冻结”，仿真器安全停止。

# 实验 6.4 冒泡排序程序测试实验——实验整体方案设计

## 1. 顶层设计方案概述

本实验旨在通过运行一个完整的**双重嵌套循环冒泡排序算法**，深度测试单周期 RV32I CPU 对复杂分支跳转指令（如 `bge`、`bltu`、`bne`）以及大批量内存读写指令（`lw`、`sw`）的连续处理能力。

### 1.1 存储布局与多通道分片设计
在实验 6.4 中，程序的存储和数据分布遵循以下原则：
*   **指令存储器 (ROM)**：加载编译生成的 `lab6.4.hex` 机器码。
*   **数据存储器 (DataRAM)**：为了支持字节/半字/字的灵活访问，DataRAM 采用 **4 片 8 位宽（单字节）RAM 并联** 的分片结构。在排序程序运行前，待排序数组的长度 $n$ 以及 $n$ 个待排序的 32 位带符号整数需分布在 4 个数据镜像文件 `lab6.4_d.hex0` ~ `lab6.4_d.hex3` 中，分别预装载至 RAM0 ~ RAM3 的对应物理地址中。
    *   地址 `0x00` 处存储 32 位整数 $n$（数组大小）。
    *   从地址 `0x04` 开始，每隔 4 个字节依次存储一个 32 位的数组元素（即 $a[1]$ 位于 `0x04`，$a[2]$ 位于 `0x08`，以此类推）。

### 1.2 冒泡排序算法执行流程
通过嵌套循环实现从小到大排序：
1.  **外层循环**：控制变量 $i$（初始化为 $n$），每轮循环结束时执行 $i = i - 1$。当 $i > 1$ 时继续，否则程序执行 `ecall` 中止。
2.  **内层循环**：控制变量 $j$（初始化为 1），通过逻辑左移指令 `slli a4, a3, 2` 计算出当前元素 $a[j]$ 的内存地址。
3.  **比较与交换 (Swap)**：
    *   使用 `lw` 读取 $a[j]$ 与 $a[j+1]$。
    *   使用 `bge` 指令进行**带符号数**比较。若 $a[j+1] \ge a[j]$，说明顺序正确，直接跳转至内循环步进处；
    *   若 $a[j+1] < a[j]$，则触发交换通路，利用 `sw` 指令将两者的值在 DataRAM 中对调。
4.  **内循环步进**：$j = j + 1$，利用 `bltu` 指令（**无符号数**比较，防止边界溢出问题）判断是否满足 $j < i$。若满足则继续内循环，否则进入外循环减量阶段。

## 2. 顶层设计模块图与数据/控制通路

单周期 CPU 顶层拓扑关系与前一实验保持一致，但数据通路在 **DataRAM 分片交互** 以及 **分支控制条件生成** 上的信号流动更加频繁。

### 2.1 4-Byte 分片式 DataRAM 内部数据流动拓扑
在执行 32 位字读写（如 `lw`、`sw`）时，数据存储器的四片 RAM 会根据地址线和控制信号进行协同，其逻辑交互关系如下：

```
                    +------------------------------------+
                    |        32-bit 内部数据总线         |
                    +------------------------------------+
                      | [31:24]   | [23:16]   | [15:8]  | [7:0]
                      v           v           v         v
                   +------+    +------+    +------+  +------+
                   | RAM3 |    | RAM2 |    | RAM1 |  | RAM0 | (单字节 RAM)
                   +------+    +------+    +------+  +------+
                      ^           ^           ^         ^
                      +-----------+-----------+---------+
                                      |
                                16位对齐地址线 (Addr[17:2])
```

*   **读数据 (Load)**：当执行 `lw a6, 0(a4)` 时，DataRAM 模块根据地址信号，同时使能 RAM0~RAM3 的片选，分别读取 1 个字节，并在内部合并拼接为 32 位字送往寄存器写回端。
*   **写数据 (Store)**：当执行 `sw a7, 0(a4)` 时，寄存器输出的 32 位数据被拆分为 4 个 8 位字节，在时钟上升沿同步写入对应的 4 片 RAM 中。

## 3. 子模块详细描述与引脚定义

针对冒泡排序的算法特征，关键子模块的控制逻辑表现如下：

### 3.1 取指部件 (IFU)
*   **详细描述**：
    在冒泡排序中，分支跳转频繁发生（包括内循环跳转 `bltu`、外循环跳转 `bne` 以及比较条件跳过 `bge`）。IFU 需要根据来自分支控制器（BandJ）计算出的 `NPCASrc` 和 `NPCBSrc` 控制信号，在每个时钟周期准确切换“顺序执行 (PC+4)”与“目标跳转 (PC+Imm)”通路。
*   **引脚定义**：
    *   同实验 6.3 基础引脚，通过 `NPCASrc` 和 `NPCBSrc` 决定下地址计算方式。其中，`halt` 端口与控制器的停机引脚相连，接收 `ecall`（`0x73`）译码信号以实现停机。

### 3.2 译码与寄存器堆部件 (IDU)
*   **详细描述**：
    本实验中频繁使用通用寄存器传递中间变量（如 $t0$ 存 $n$，$a1$ 存常数 $1$，$a2$ 存 $i$，$a3$ 存 $j$，$a4$ 存 $a[j]$ 地址，$a6/a7$ 暂存待比较数据）。立即数生成器依据 `ExtOp` 产生 `slli` 移位量、`lw/sw` 偏移量以及分支跳转偏移量。
*   **引脚定义**：
    *   同实验 6.3。移位指令 `slli` 的立即数在译码阶段被正确提炼并送入 ALU 的输入 B 端。

### 3.3 控制器 (Controller)
*   **详细描述**：
    控制器是冒泡排序的核心指挥中心，对多条关键指令产生对应的控制使能：
    *   `slli`：识别为 `I_arith`（操作码 `0010011`），`ExtOp = 000`，`ALUASrc = 0`，`ALUBSrc = 10`（选择立即数），`ALUctr = 0001`（逻辑左移）。
    *   `bge` / `bltu` / `bne`：识别为 `B_instr`（操作码 `1100011`），根据 `funct3` 区分。`RegWr = 0`，`ALUASrc = 0`，`ALUBSrc = 00`，`ALUctr = 0010/0011`（减法/无符号比较），`BandJ` 产生相应的跳转判定使能。
    *   `ecall`：识别为停机指令，拉高 `Halt`。
*   **引脚定义**：
    *   输入输出端口与 6.3 保持一致。

### 3.4 算术逻辑单元 (ALU)
*   **详细描述**：
    除了进行加减法，ALU 本次需要支持**无符号比较**（用于 `bltu` 判断循环边界）和**有符号比较**（用于 `bge` 比较数组元素大小）。
*   **引脚定义**：
    *   输入与 6.3 保持一致，通过 `ALUctr` 指定有符号减（`1000`）、无符号比较（`0011`）或逻辑左移（`0001`）。

### 3.5 数据存储器 (DataRAM)
*   **详细描述**：
    必须完整实现 32 位“字（Word）”读写控制。当 `MemOp = 000` 且 `MemWr = 1` 时，内部写使能分配器同时向 RAM0~RAM3 发送写有效信号。
*   **引脚定义**：
    *   `Addr` [32位，输入]、`Din` [32位，输入]、`WE` [1位，输入]、`CLK` [1位，输入]、`MemOp` [3位，输入]、`Dout` [32位，输出]。

## 4. 数据交换（Swap）核心通路详细定义

冒泡排序中，当相邻两个元素逆序（$a[j] > a[j+1]$）时，需要通过交换通路进行数据对调。这一过程由 4 条连续指令协作完成，其底层硬件通路流动及控制信号配合定义如下：

```
 步骤 1: 读出 a[j] (lw a6, 0(a4))
   控制信号: RegWr=1, ALUASrc=0, ALUBSrc=10, ALUctr=ADD, MemtoReg=1, MemWr=0, MemOp=Word
   物理通路: [R[a4] + 0] -> ALU -> DataRAM 地址端 -> DataRAM 读出 32位字 -> BusW -> 写入 R[a6]

 步骤 2: 读出 a[j+1] (lw a7, 4(a4))
   控制信号: RegWr=1, ALUASrc=0, ALUBSrc=10, ALUctr=ADD, MemtoReg=1, MemWr=0, MemOp=Word
   物理通路: [R[a4] + 4] -> ALU -> DataRAM 地址端 -> DataRAM 读出 32位字 -> BusW -> 写入 R[a7]

 步骤 3: 写入 a[j] 的位置 (sw a7, 0(a4))
   控制信号: RegWr=0, ALUASrc=0, ALUBSrc=10, ALUctr=ADD, MemWr=1, MemOp=Word
   物理通路: [R[a4] + 0] -> ALU -> DataRAM 地址端; 寄存器输出 R[a7] -> Din -> 写入 DataRAM (分片 RAM0~3 协同写入)

 步骤 4: 写入 a[j+1] 的位置 (sw a6, 4(a4))
   控制信号: RegWr=0, ALUASrc=0, ALUBSrc=10, ALUctr=ADD, MemWr=1, MemOp=Word
   物理通路: [R[a4] + 4] -> ALU -> DataRAM 地址端; 寄存器输出 R[a6] -> Din -> 写入 DataRAM (分片 RAM0~3 协同写入)
```

通过这四个周期的数据搬运，逆序数据在物理存储单元中完成了位置对调，硬件通路完美支持了冒泡排序算法的物理层交换需求。

# 思考题
## 1. 如何在单CPU 上实现多任务处理，例如同时执行计算累加和与数据排序两个程序，阐述思路。
在单个处理器上实现多任务处理，核心思想是分时复用，让操作系统在不同任务之间快速切换，以此制造出“同时执行”的假象。
- **引入时钟中断与时间片**：在硬件上引入定时器部件，当定时器递增到设定值时，向处理器触发一个硬件中断信号。以此为基准，将处理器的运行时间划分为微小的时间片。
*   **上下文保存与切换**：当一个程序（如累加和）的时间片用完并触发时钟中断时，处理器暂停该程序的执行。硬件或操作系统内核需要将当前程序的“上下文状态”——包括程序计数器、所有通用寄存器的值、状态寄存器等，保存到特定的内存区域（通常是该任务的内核栈）。
*   **任务调度与上下文恢复**：调度程序根据特定的调度算法，选择下一个要运行的任务（如数据排序），从内存中恢复其先前保存的上下文状态，将其写入物理寄存器和程序计数器中。随后，处理器继续执行排序程序。
*   **结果**：由于时间片通常只有几毫秒甚至更短，在人类宏观感官上，两个任务看起来就像是在并行。

## 2. 在 CPU 的基础上，如何实现键盘输入、TTY 输出部件等输入输出设备的数据访问，构建完整的计算机系统。
可以通过如下方案来实现处理器与外部设备的数据交互，进而构建完整的计算机系统。

*   **统一编址与地址译码**：采用内存映射输入输出。在物理地址空间中，划出一部分特定范围的地址（例如高地址区）作为外设寄存器的映射地址，而不将其分配给物理内存。通过扩展系统总线上的地址译码器，当处理器发出的物理地址落在外设空间时，译码器关闭物理内存的片选信号，转而开启对应外设控制器的片选信号。
*   **外设控制器与接口寄存器**：每个外设都需要一个控制芯片。例如，键盘控制器包含“数据寄存器”和“状态寄存器”。当用户按下按键，按键的扫描码存入键盘数据寄存器，同时状态寄存器的“准备好”标志位置位。
*   **数据访问方式**：
    *   **轮询方式**：处理器通过循环读取外设的状态寄存器，直到发现准备好标志有效，再读取数据寄存器。这种方式编程简单，但处理器等待时间长、效率较低。
    *   **中断驱动方式**：当外设准备好数据后，主动向处理器发送中断请求。处理器在每条指令执行结束时检测该请求，若有，则暂停当前程序，跳转到中断服务程序读取外设数据，处理完后再返回。这种方式效率较高。

## 3. 阐述如何在单周期CPU 基础上实现多周期CPU
分析：单周期CPU的CPI为1，时钟周期由最长的`lw`指令决定；而多周期CPU将一条指令拆分成若干步，每一步用一个时钟周期完成。
实现思路如下：
*   **划分执行步骤**：将单周期中一气呵成的指令执行过程（指一条指令花费一个时钟周期完成），划分成多个更小、耗时更均衡的阶段（取指、译码、执行、访存、写回）。
*   **引入暂存寄存器**：在各个阶段之间加入不需要程序员显式可见的暂存寄存器（如指令寄存器、数据暂存器、算术逻辑单元输出暂存器等），用于锁存上一个阶段的计算结果，供下一个时钟周期使用。
*   **设计状态控制器**：多周期处理器的控制器不能再是纯组合逻辑，必须引入一个有限状态机。有限状态机根据当前指令的操作码和当前所处的时钟周期状态，动态输出对应阶段的控制信号。
*   **复用硬件资源**：因为不同步骤在不同的时钟周期内发生，原本在单周期中必须分开的硬件可以合并。例如，不再需要独立的程序计数器加法器，取指阶段和算术逻辑单元可以共用同一个算术逻辑单元；指令存储器和数据存储器也可以合并为同一个统一存储器。

## 4. 阐述如何在单周期CPU 基础上实现流水线 CPU？
流水线处理器的设计是为了在多周期分步执行的基础上，实现指令级并行。其设计和转换思路如下：
*   **插入流水线寄存器**：在单周期的各个核心阶段之间，强行插入流水线寄存器，把组合逻辑数据通路切成5段：取指、译码、执行、访存、写回。流水线寄存器在每个时钟上升沿将本级的结果传递给下一级，使得每个时钟周期都可以有一条新指令进入流水线，多条指令在不同的流水段同时执行。
*   **控制信号随指令流动**：控制器在译码阶段一次性生成该指令在后续各级所需的所有控制信号，但这些控制信号不能立刻作用于全局，而是必须被锁存进段间寄存器，伴随着指令的数据和地址一起，逐级在流水线寄存器中向下传递。
*   **处理流水线冒险**：由于多条指令并行，需要增加专门的硬件逻辑来解决以下三类冒险：
    *   **结构冒险**：多个流水段同时访问同一个硬件。解决思路是将指令存储器和数据存储器物理分离，或对寄存器堆采用前半周期写、后半周期读的设计。
    *   **数据冒险**：后一条指令需要前一条尚未写回的计算结果。解决思路是使用转发/旁路，将执行阶段或访存阶段的结果直接反馈回算术逻辑单元的输入端；若无法前推，则需要通过控制逻辑进行流水线暂停（插入bubble）来等待数据。
    *   **控制冒险**：分支跳转指令导致后续预取指令失效。解决思路是引入分支预测技术（静态或动态预测），若预测失败则冲刷已进入流水线的前几条错误指令（将控制信号清零）。
