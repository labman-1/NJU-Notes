# 7
分析：至多支持64种操作，$2^6=64$，故操作码占6位。每个操作数支持4种寻址方式，故每个操作数需要2位来标识寻址方式。一共有8个寄存器，故每个寄存器编号占3位。题目给出了存储器地址位数和立即数均为 16 位。
字长为16位，指令长度必须为16位的整数倍。

根据上面的分析，我们定义 **Word 1（首字）** 格式如下：
`| OP (6位) | M1 (2位) | R1 (3位) | M2 (2位) | R2 (3位) |` (总计16位)

*   **$M_i$ 编码约定**：`00`: R, `01`: I, `10`: S, `11`: X
*   **$R_i$**：对应的通用寄存器编号。

#### 7 种指令格式详细设计表

| 指令类型 | 指令结构 (Word 1 | Word 2 | Word 3) | 总长度 | 访存次数 (取指+数据) | 备注 |
| :--- | :--- | :--- | :--- | :--- | :--- | :--- |
| **RR型** | `OP M1 R1 M2 R2` | - | - | 16位 | 1 + 0 = **1次** | 全在寄存器里 |
| **RI型** | `OP M1 R1 M2 R2` | **立即数 (16位)** | - | 32位 | 2 + 0 = **2次** | 指令占2字，无数据访存 |
| **RS型** | `OP M1 R1 M2 R2` | - | - | 16位 | 1 + 1 = **2次** | S 需要去内存读/写一次 |
| **RX型** | `OP M1 R1 M2 R2` | **位移量 (16位)** | - | 32位 | 2 + 1 = **3次** | 指令2字 + X数据访存1次 |
| **XI型** | `OP M1 R1 M2 R2` | **位移量 (16位)** | **立即数 (16位)** | 48位 | 3 + 1 = **4次** | 指令3字 + X数据访存1次 |
| **SI型** | `OP M1 R1 M2 R2` | **立即数 (16位)** | - | 32位 | 2 + 1 = **3次** | 指令2字 + S数据访存1次 |
| **SS型** | `OP M1 R1 M2 R2` | - | - | 16位 | 1 + 2 = **3次** | 指令1字 + 两个S数据访存 |
