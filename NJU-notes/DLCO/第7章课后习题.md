分析：至多支持64种操作，$2^6=64$，故操作码占6位。每个操作数支持4种寻址方式，故每个操作数需要2位来标识寻址方式。一共有8个寄存器，故每个寄存器编号占3位。题目给出了存储器地址位数和立即数均为 16 位。
字长为16位，指令长度必须为16位的整数倍。

根据上面的分析，我们定义Word 1（首字）格式如下：
`| OP (6位) | M1 (2位) | R1 (3位) | M2 (2位) | R2 (3位) |` (总计16位)
*   $M_i$ 编码约定：`00`: R, `01`: I, `10`: S, `11`: X
*   **$R_i$**：对应的通用寄存器编号。
* 
 7 种指令格式详细设计表如下

| :------ | :--------------- | :------------ | :------------ | :---- | :------------- | :------------- |
| **RR型** | `OP M1 R1 M2 R2` | -             | -             | 16位   | 1 + 0 = **1次** | 全在寄存器里         |
| **RI型** | `OP M1 R1 M2 R2` | **立即数 (16位)** | -             | 32位   | 2 + 0 = **2次** | 指令占2字，无数据访存    |

# 8
(1)操作码占4位，故该指令系统最多可有$2^4=16$条指令。
通用寄存器编号占3位，故至多有8个通用寄存器。
地址空间大小为128KB，按字编址，故有$64K=2^16$个存储单元，则地址数为16位，MAR至少为16位。由于字长为16位，MDR至少为16位。
(2)
鉴于地址位数与字长都设计为16位，程序计数器（PC）和通用寄存器自然也是16位。两个16位数值相加后，结果仍为16位，因此转移目标地址的宽度同样是16位。这便使得转移操作可以覆盖整个地址空间，其目标地址范围可从0000H一直到FFFFH。
(3)
add对应操作码为0010B，(R4)对应寻址方式字段Ms为001B，R4编号为100B；(R5)+对应寻址方式字段Md为010B，R5编号为101B。故该汇编语句对应的机器码为0010 001 100 010 101B，用十六进制表示为2315H。
分析执行过程：
1.  取源操作数：`(R4)` 是寄存器间接寻址。$R4 = 1234H$，则源操作数 = $M[1234H] = 5678H$。
2.  取目的操作数：`(R5)+` 是寄存器间接寻址。$R5 = 5678H$，则目的操作数 = $M[5678H] = 1234H$。
3.  加法计算：$5678H + 1234H = 68ACH$。
4.  写回结果：结果存回目的操作数的有效地址，即 $M[5678H]$  变为 $68ACH$。
5.  自增操作：执行完后 $R5$ 自增。由于是按字编址，$R5 = 5678H + 1 = 5679H$。
所以寄存器R5会被改变，内容变为5679H；编号为5678H的存储单元中的内容变为68ACH。

# 10
设 `imm20` 是 `auipc` 实际使用的 20 位值，`imm12` 是 `offset[11:0]`。jalr会对操作数`imm12`进行符号扩展。

我们需要满足：

`(PC0 + (imm20 << 12)) + sign_extend(imm12) = PC0 + offset`

消去 `PC0`，等价于：

**`(imm20 << 12) + sign_extend(imm12) = offset`**

根据有符号扩展的定义：

`sign_extend(imm12) = imm12 - (imm12[11] ? 2^12 : 0)`  
等价于 
`sign_extend(imm12) = imm12 - offset[11] * 2^12`

代入上式：

`(imm20 << 12) + imm12 - offset[11] * 2^12 = offset`  
→ `(imm20 - offset[11]) << 12 + imm12 = offset`

而 `imm12` 就是 `offset` 的低 12 位，`offset` 可以写成 `(offset[31:12] << 12) + offset[11:0]`。比较等式两边，可得：

`imm20 - offset[11] = offset[31:12]`

即：

**`imm20 = offset[31:12] + offset[11]`**

这就是 `auipc` 立即数需要加上 `offset[11]` 的根本原因：补偿 `jalr` 时符号扩展引入的 `-2^12` 。

# 17
beq指令是RV32I中的有条件分支指令，`beq rs1, rs2, label`表示比较rs1和rs2的值，若相等则程序跳到label处执行；若不等则继续执行当前指令的下一条指令（PC+4）. 寻址方式：采用PC相对寻址，指令中存储的为偏移量$offset=imm[12:1]乘2$，目标地址为PC+offset。

为什么会出错：beq是B-type格式指令。在32位的指令字中，只有12位留给立即数（offset）。这12位立即数是有符号数，且末尾隐含一个0，故表示的真实范围为$[-2^{12},2^{12}-2]$字节。
若代码段中here和there表示的地址之差超过上述范围，汇编器会发现12位的立即数塞不下这个偏移量，从而报错。

如何修改：

```assembly
here: bne t0, t2, skip_jump  # 1. 如果不相等，跳到 skip_jump（近距离跳）
      j   there              # 2. 如果走到这，说明相等。用 j 指令跳到 there（远距离跳）
      ....                   # 这里的 .... 原本的内容
skip_jump:
      ....                   # 紧跟 here 之后的其他代码
      ....
there: addi t1, a0, 4        # 目标位置
```
