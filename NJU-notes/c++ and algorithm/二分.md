## 二分查找
二分查找是用来在一个有序数组中查找某一元素的算法。
```c++
int binary_search(int start, int end, int key) { int ret = -1; // 未搜索到数据返回-1下标 
int mid; 
while (start <= end) {
mid = start + ((end - start) >> 1); // 直接平均可能会溢出，所以用这个算法 
if (arr[mid] < key) start = mid + 1; 
else if (arr[mid] > key) end = mid - 1;
 else { // 最后检测相等是因为多数搜索情况不是大于就是小于 
ret = mid; break; 
} 
} 
return ret; // 单一出口 
}
```
## 最大值最小化
若一个数组中的左侧或者右侧均满足某一种条件，而另一侧都不满足，也可视为有序。二分搜索法可以用来查找满足某种条件的最大（最小）值。
满足某种条件的最大值的最小可能情况（最大值最小化），首先的想法是从小到大枚举这个作为答案的「最大值」，然后去判断是否合法．若答案单调，就可以使用二分搜索法来更快地找到答案．因此，要想使用二分搜索法来解这种「最大值最小化」的题目，需要满足以下三个条件：

1. 答案在一个固定区间内；
2. 可能查找一个符合条件的值不是很容易，但是要求能比较容易地判断某个值是否是符合条件的；
3. 可行解对于区间满足一定的单调性．换言之，如果 𝑥![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "x") 是符合条件的，那么有 𝑥 +1![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "x + 1") 或者 𝑥 −1![](data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7 "x - 1") 也符合条件．（这样下来就满足了上面提到的单调性）

当然，最小值最大化是同理的．

### 什么题能用二分答案？

在做题时，如果题目让你求 **“最大的最小值”** 或者 **“最小的最大值”**，或者求 **“满足条件的最大/小 K 值”**，你的二分雷达就要响了。🚨

假设我们要解决这样一个问题：有 $N$ 根绳子，我要切出 $K$ 段**同样长度**的绳子，问这 $K$ 段绳子**最长**能有多长？

*   **正向思考**：直接算这个长度是多少，好像很难，要考虑各种切法。
*   **逆向思考（二分答案）**：
    如果我问你：“能不能切出长度为 $10$ 的 $K$ 段？” 你能很容易回答我“能”还是“不能”吗？
    如果长度 $10$ 能切出来，那长度 $5$ 一定能切出来吗？
    如果长度 $10$ 切不出来，那长度 $20$ 还能切出来吗？

**总结：**
这就是二分答案的核心——把**求解问题** （ 计算 $X$ 是多少）转化为**判定问题**（给定 $X$，判断它是否合法）。
这个负责判断的函数，我们通常叫它 **`check(mid)`**。

### 代码实现的“生死线”

二分答案最容易写挂的地方，不是 `check` 函数，而是那个该死的 `while` 循环边界和 `mid` 的计算。很多同学写完发现死循环了，或者答案总是差 1。

这里有两套**必须背诵**的模板，分别对应两种情况。请拿出小本本记下来！📝

#### 模板一：找满足条件的“最小值”（左边界）
通常用于：求“最小花费”、“最短时间”等。
也就是：
$$ \dots \text{False}, \text{False}, \mathbf{\text{True}}, \text{True}, \text{True} \dots $$ 
我们要找那个绿色的第一个 True。

```cpp
// 区间 [l, r]
while (l < r) {
    int mid = l + (r - l) / 2; // 向下取整，即 (l+r)/2
    if (check(mid)) {
        r = mid;     // mid 可能是答案，也可能更小的才是，所以保留 mid，尝试往左缩
    } else {
        l = mid + 1; // mid 肯定不行，答案一定在 mid 右边
    }
}
// 循环结束时 l == r，就是答案
return l;
```

#### 模板二：找满足条件的“最大值”（右边界）
通常用于：求“最大长度”、“最多数量”等。
也就是：
$$ \dots \text{True}, \text{True}, \mathbf{\text{True}}, \text{False}, \text{False} \dots $$
我们要找那个红色的最后一个 True。

```cpp
// 区间 [l, r]
while (l < r) {
    int mid = l + (r - l + 1) / 2; // 【注意】这里必须 +1，向上取整！
    if (check(mid)) {
        l = mid;     // mid 可能是最大值，也可能更大的才是，保留 mid，尝试往右缩
    } else {
        r = mid - 1; // mid 都不行，更大的肯定不行，答案在左边
    }
}
return l;
```

**学姐灵魂拷问 3：**
请仔细看模板二的 `mid` 计算：`int mid = l + (r - l + 1) / 2;`。
如果你不加这个 `+1`，也就是写成了 `(l+r)/2`。
当区间只剩下两个数，比如 `l=2, r=3` 时：
1. `mid` 会变成多少？
2. 如果 `check(mid)` 成功了，执行 `l = mid`，`l` 会变成多少？
3. 下一次循环，区间是 `[?, ?]`？程序会发生什么？（这是造成死循环的罪魁祸首！）

### 总结与实战

**二分答案三步走：**
1.  **定边界**：确定答案可能的最小值 $L$ 和最大值 $R$。
2.  **写 Check**：写一个 `bool check(mid)` 函数，判断值为 `mid` 时是否满足题目条件（通常是一个贪心或简单的遍历）。
3.  **套模板**：根据你是要找“最大”还是“最小”，套用上面对应的模板。

### [P2678 [NOIP 2015 提高组] 跳石头 - 洛谷](https://www.luogu.com.cn/problem/P2678)
```c++
#include<bits/stdc++.h>
using namespace std;
int l, n,m;
int rock[50005];
//mid 表示预期最短路径最大值，check检查是否合法
bool check(int mid) {
	int removeNum = 0, currentPos = 0,nextRock=1;
	while ( nextRock <= n+1 ) {
		int path = rock[nextRock] - rock[currentPos];
		if (path < mid) {
			removeNum++;
		}
		else {
			currentPos=nextRock;
		}
		if (removeNum > m) {
			return false;
		}
		nextRock++;
	}
	return true;
}
int main() {
	ios::sync_with_stdio(false);cin.tie(0);
	cin >> l >> n >> m;
	for (int i = 1;i <= n;i++) {
		cin >> rock[i];
	}
	rock[0] = 0,rock[n+1]=l;
	int le = 1,right = l;
	while (le < right) {
		int mid = le + ((right - le + 1) >> 1);
		if (check(mid)) {
			le = mid;
		}
		else {
			right = mid-1;
		}
	}
	cout << le;
	return 0;
}
```
