
###🌟 破局：从 LCS 变成 LIS ($O(N \log N)$)

**P1439** 这道题有一个**极其重要**的条件，你可能忽略了：
> **这两个序列都是 $1 \sim n$ 的全排列。**

这意味着：序列里没有重复数字，且两个序列包含的数字集合完全一样，只是顺序不同。

**核心思路（骚操作）：**
我们可以把 **“最长公共子序列 (LCS)”** 问题，转化成 **“最长上升子序列 (LIS)”** 问题来解！

#### 1. 转化原理 (Mapping)
假设：
*   A: `3 2 1 4 5`
*   B: `1 2 3 4 5`

我们的目标是找到 A 和 B 都有的子序列。
因为 B 是 `1 2 3 4 5`（有序的），实际上我们就是在 A 里找一个**“按 1,2,3... 顺序排列”**的子序列。

**更通用的情况：**
*   A: `3 2 1 4 5`
*   B: `1 2 3 4 5` (假设这是另一个乱序序列，比如 B 是 `2 5 3 1 4`)

**转化步骤：**
1.  **以 A 为标准**：我们将 A 中的数字重新编号。
    *   让 `3` 变成 `a` (第1个)
    *   让 `2` 变成 `b` (第2个)
    *   让 `1` 变成 `c` (第3个)
    *   ...
    *   那么 A 就变成了 `a b c d e` (严格递增)。

2.  **映射 B**：用同样的规则把 B 里的数字换掉。
    *   B 中出现的 `3` 换成 `a` (1)，`2` 换成 `b` (2)...
    *   我们要找 B 和 A 的公共子序列。
    *   因为 A 已经是严格递增的了，所以我们只要在 **映射后的 B** 里找一个 **严格递增** 的子序列，它就一定是公共的！

#### 2. 举个栗子 (手动模拟)

*   **A**: `3 2 1 4 5`
*   **B**: `1 2 3 4 5`

**第一步：建立映射 (Map)**
记录 A 中每个数字出现的位置：
*   `pos[3] = 1`
*   `pos[2] = 2`
*   `pos[1] = 3`
*   `pos[4] = 4`
*   `pos[5] = 5`

**第二步：转换 B**
把 B 中的每个数字，替换成它在 A 中的位置下标：
*   B 原文: `1  2  3  4  5`
*   B 映射: `3  2  1  4  5` (这是 B 中每个数在 A 里对应的下标)

**第三步：求 LIS**
现在问题变成了：求序列 `[3, 2, 1, 4, 5]` 的 **最长上升子序列**。
*   LIS 是 `1, 4, 5` (长度 3)。
*   还原回原数字：A 的第1个、第4个、第5个 $\to$ `3, 4, 5`？不对，是下标的 LIS。
*   其实我们只需要**长度**。长度就是 3。

### 🚀 进阶：如何写 $O(N \log N)$ 的 LIS？

既然转化成了 LIS，如果还用 $O(N^2)$ 的 DP，还是会超时。你需要 **贪心 + 二分查找** 的写法。

**算法逻辑：**
我们维护一个数组 `low`（或者叫 `q`），`low[i]` 表示 **“长度为 i 的上升子序列，结尾最小是多少”**。
*   **贪心策略**：如果我想让子序列尽可能长，那我肯定希望结尾的数越小越好，这样后面才容易接上更大的数。

**操作流程：**
遍历数组中的每个数 `x`：
1.  如果 `x` 比 `low` 数组最后一个数还大 $\to$ **直接接在后面**，LIS 长度 +1。
2.  如果 `x` 比 `low` 最后一个数小 $\to$ **在 `low` 里找一个比 `x` 大的最小数，把它替换成 `x`**。
    *   *潜台词：虽然长度没变，但我把结尾变小了，潜力变大了！*
    *   查找替换这一步用 `lower_bound` (二分查找)。

### 📜 满分代码模板 (P1439 AC版)

这个代码包含了**映射转化**和**二分LIS**两个知识点，含金量极高。

```cpp
#include <bits/stdc++.h>
using namespace std;

const int N = 100005;
int a[N], b[N];
int map_a[N]; // 记录 A 中每个数值对应的下标
int new_b[N]; // B 转化后的数组
int low[N];   // 贪心数组：长度为 i 的 LIS 结尾最小值

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);

    int n;
    cin >> n;

    // 1. 读入 A 并建立映射
    for (int i = 1; i <= n; i++) {
        cin >> a[i];
        map_a[a[i]] = i; // 数值 a[i] 在 A 中的位置是 i
    }

    // 2. 读入 B 并转化为下标序列
    for (int i = 1; i <= n; i++) {
        cin >> b[i];
        new_b[i] = map_a[b[i]]; // 获取 b[i] 在 A 里的位置
    }

    // 3. 求 new_b 的 LIS (O(N log N) 做法)
    int len = 1;
    low[1] = new_b[1]; // 初始化：长度为1的子序列结尾是第一个数

    for (int i = 2; i <= n; i++) {
        if (new_b[i] > low[len]) {
            // 如果当前数比最长序列的结尾还大，直接接上
            low[++len] = new_b[i];
        } else {
            // 否则，用二分查找在 low 中找到第一个 >= new_b[i] 的位置，替换掉
            // lower_bound 返回的是迭代器，减去 begin 得到下标
            int pos = lower_bound(low + 1, low + len + 1, new_b[i]) - low;
            low[pos] = new_b[i];
        }
    }

    cout << len;
    return 0;
}
```

### 总结
1.  **普通 LCS**：$O(N^2)$，只能做 $N \le 5000$。
2.  **排列 LCS**：转化为 **LIS** 问题。
3.  **大快朵颐 LIS**：用 **贪心+二分** 做到 $O(N \log N)$，秒杀 $10^5$。
