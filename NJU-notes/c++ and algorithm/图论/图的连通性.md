**场景：** 分析图的结构，有没有环？谁是关键节点？
**机试地位：** ⭐⭐（属于高阶题，通常作为压轴）
##  Tarjan 算法 (塔扬)
*   **人设**：**“时间管理的祖师爷”**。
*   **一句话原理**：利用 DFS 的访问顺序（时间戳 `dfn`）和能回溯到的最早祖先（`low`），来判断图中哪里成了环（强连通分量），或者哪条边一断图就裂开了（桥）。
* 学弟/学妹，**Tarjan 算法**是图论中的“重型核武器”。它由 Robert Tarjan 发明，专门用来解决**强连通分量 (SCC)** 的问题。

在算法竞赛中，很多人觉得 Tarjan 难，是因为它的逻辑像“绕口令”。学姐带你用**“深搜+栈”的哲学**把它拆解开，保证你能看懂。

---

### 1. 核心概念：什么是强连通分量 (SCC)？
在**有向图**中，如果两个点 $u$ 和 $v$ 能够互相到达（$u \to v$ 且 $v \to u$），那么它们就属于同一个“强连通分量”。

**Tarjan 的目标**：把一张复杂的有向图，压缩成一个由若干个 SCC 组成的 **DAG（有向无环图）**。

---

### 2. Tarjan 的“秘密武器” (两个数组)

你需要维护两个关键的数组，这是 Tarjan 的“大脑”：

1.  **`dfn[u]` (Discovery Time)**：时间戳。表示点 `u` 是在第几个被 DFS 访问到的。一旦盖章，绝不更改。
2.  **`low[u]` (Lowest Link)**：**这是最难理解的！** 它表示从 `u` 出发，通过 DFS 树上的边和**一条**回边（Back-edge），能够追溯到的最早的祖先节点的 `dfn` 值。

---

### 3. 三步走：Tarjan 算法的执行逻辑

我们可以把算法看作一个“回溯打标签”的过程：

1.  **DFS 盖章**：进入点 `u`，`dfn[u] = low[u] = ++timer`。同时，把 `u` 压入一个 **栈**（这个栈记录了当前搜索路径上的点）。
2.  **深入遍历**：遍历 `u` 的所有邻居 `v`：
    *   **如果 `v` 没被访问过**：继续 `dfs(v)`。回溯后，用 `low[v]` 更新 `low[u]`（即 `low[u] = min(low[u], low[v])`）。
    *   **如果 `v` 已经在栈里**：说明 `v` 是 `u` 的祖先，`v` 构成了环，用 `dfn[v]` 更新 `low[u]`（即 `low[u] = min(low[u], dfn[v])`）。
3.  **发现 SCC**：
    当 `dfn[u] == low[u]` 时，说明 `u` 是这个 SCC 的“龙头老大”（根节点）。此时，从栈里不断弹出元素，直到弹出 `u` 为止。这些弹出来的点，共同构成了一个强连通分量。

---

### 4. 为什么要用“栈”？

栈的意义在于：**它维护了当前 DFS 路径上的所有点**。
当 `dfn[u] == low[u]` 时，栈顶到 `u` 之间的所有点，就是刚才那个“环”或者“强连通区域”里还没被归类的点。把它们抠出来，就是我们要找的一个分量。

---

### 5. 为什么叫“缩点”？（Tarjan 的价值）

这是 Tarjan 最常用的场景：
如果你把每一个 SCC 里的点都看作一个“超级节点”，那么原图就变成了一个 **DAG**。
*   **好处**：DAG 上面可以跑 DP！
*   很多复杂的图论题，只要一缩点，立刻就能变成简单的 DP 问题。

---

### 💡 工业级/竞赛级代码模板 (建议背诵)

```cpp
int dfn[MAXN], low[MAXN], timer;
stack<int> s;
bool inStack[MAXN]; // 标记点是否在栈里
int scc[MAXN], scc_cnt; // 记录每个点属于哪个SCC

void tarjan(int u) {
    dfn[u] = low[u] = ++timer;
    s.push(u);
    inStack[u] = true;

    for (int v : adj[u]) {
        if (!dfn[v]) {
            tarjan(v);
            low[u] = min(low[u], low[v]);
        } else if (inStack[v]) {
            low[u] = min(low[u], dfn[v]);
        }
    }

    if (dfn[u] == low[u]) { // 发现老大，弹出栈
        scc_cnt++;
        while (true) {
            int v = s.top(); s.pop();
            inStack[v] = false;
            scc[v] = scc_cnt;
            if (u == v) break;
        }
    }
}
```

1.  **逻辑辨析**：`low[u] = min(low[u], low[v])` 和 `low[u] = min(low[u], dfn[v])` 这两行代码的本质区别是什么？（提示：一个是用子节点的“追溯能力”更新自己，一个是直接用祖先的“时间戳”更新自己）。
2.  **逻辑推演**：如果一个图本身就是一个环 `1->2->3->1`，那么执行 Tarjan 后，最终 `scc_cnt` 是多少？所有节点的 `low` 值最终会变成多少？
