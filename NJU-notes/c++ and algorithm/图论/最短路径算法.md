**场景**：地图导航，从A到B怎么走最快？
## Floyd算法
- **“上帝视角的中转站”
- **一句话原理**：我想从i到j，能不能通过中间点k中转一下，使路径变短？（i->k->j比i->j短吗？）
- **特征：
	- **核心：3个for循环
	- **适用**：点很少（N<=500），求**任意两点**间的最短路径。
	- **记忆点**：k（中转点）在最外层循环。
### 一、 Floyd 是干什么用的？

*   **Dijkstra**：解决 **“单源最短路”**。
    *   即：计算 **起点 A** 到 B、C、D... 所有人的最短距离。
    *   *就像我要去送快递，从我家出发去所有人那里。*
*   **Floyd**：解决 **“多源最短路”**（All-Pairs Shortest Path）。
    *   即：计算 **任意两个人**（A到B，C到D，E到A...）之间的最短距离。
    *   *就像上帝在看地图，他要知道地图上任意两个城市之间的最短路线。*

### 二、 核心原理：中转站思维 (DP)

Floyd 的核心思想不是“贪心”，而是 **“尝试每一个点做中转站”**。

假设你想从 **南京 ($i$)** 去 **东京 ($j$)**：
1.  **直飞**：距离是 `dist[i][j]`。
2.  **中转**：如果先去 **上海 ($k$)** 中转一下，路径变成 `南京 -> 上海 -> 东京`。
    *   如果 `南京->上海` ($dist[i][k]$) + `上海->东京` ($dist[k][j]$) < `直飞距离` ($dist[i][j]$)。
    *   那就果断选择中转！更新最短距离。

**DP 状态转移方程：**
$$dist[i][j] = \min(dist[i][j], \ dist[i][k] + dist[k][j])$$

---

### 三、 代码模板（背诵全文，仅 4 行）

Floyd 的代码是所有图论算法里最短的，短到令人发指。它就是 **3 个 `for` 循环**。

**⚠️ 唯一的、绝对不能错的重点：**
**中转点 $k$ 必须写在最外层循环！** （这是 99% 初学者会踩的坑）

```cpp
// d[i][j] 存的是从 i 到 j 的最短距离
// 初始化：d[i][i] = 0，其他全是 INF，如果有边直连则为边权

// 第一层循环：枚举中转点 k (阶段)
for (int k = 1; k <= n; k++) {
    // 第二层循环：枚举起点 i
    for (int i = 1; i <= n; i++) {
        // 第三层循环：枚举终点 j
        for (int j = 1; j <= n; j++) {
            // 核心状态转移：
            // 试图让 k 介入 i 和 j 之间
            if (d[i][k] != INF && d[k][j] != INF) { // 防止 INF+INF 溢出
                 d[i][j] = min(d[i][j], d[i][k] + d[k][j]);
            }
        }
    }
}
```

### 四、 为什么要学它？（优缺点分析）

你可能会问：*“我跑 N 遍 Dijkstra 不也能求出任意两点的距离吗？为啥要学 Floyd？”*

**优点：**
1.  **好写**：考场上如果数据范围小，写 Floyd 只需要 30 秒，写 Dijkstra 要 5 分钟。
2.  **能处理负权边**：Dijkstra 遇到负权边直接歇菜，Floyd 可以处理（只要没有负权环）。
3.  **机试捡分神器**：回忆版机试里提到“写出 Floyd 能拿 30 分的基础分”，指的就是当题目太难你不会正解时，看数据范围如果 $N \le 300$，直接扔一个 Floyd 上去，把部分分拿满。

**缺点（致命）：**
1.  **太慢了**：时间复杂度 $O(N^3)$。
    *   $N=100 \to 10^6$ 次运算（毫秒级）。
    *   $N=1000 \to 10^9$ 次运算（**超时 TLE**）。
    *   **Dijkstra (堆优化)** 是 $O(M \log N)$，跑 $N=10^5$ 都没问题。

**总结：** 看到数据范围 $N \le 300$ (或 $200$)，无脑 Floyd；看到 $N > 1000$，想都别想，切 Dijkstra。

### 五、 实战练习

我们还是拿 **洛谷 P4779** 或者随便一个图论题的数据结构来举例。
但 Floyd 通常使用 **邻接矩阵** (`int dist[N][N]`) 来存图，而不是邻接表。

**代码骨架：**

```cpp
#include <bits/stdc++.h>
using namespace std;

const int INF = 0x3f3f3f3f;
const int N = 305; // Floyd 一般只用于 N 较小的情况

int dist[N][N]; // 邻接矩阵
int n, m;

int main() {
    cin >> n >> m;

    // 1. 初始化
    // 自己到自己是 0，到别人一开始全是无穷大
    for (int i = 1; i <= n; i++) {
        for (int j = 1; j <= n; j++) {
            if (i == j) dist[i][j] = 0;
            else dist[i][j] = INF;
        }
    }

    // 2. 读入边
    for (int i = 0; i < m; i++) {
        int u, v, w;
        cin >> u >> v >> w;
        // 注意：如果有重边，要取最小的
        dist[u][v] = min(dist[u][v], w); 
        // 如果是无向图：
        // dist[v][u] = min(dist[v][u], w);
    }

    // 3. Floyd 核心 (背下来！)
    // k 必须在外层！
    for (int k = 1; k <= n; k++) {
        for (int i = 1; i <= n; i++) {
            for (int j = 1; j <= n; j++) {
                if (dist[i][k] != INF && dist[k][j] != INF) {
                     dist[i][j] = min(dist[i][j], dist[i][k] + dist[k][j]);
                }
            }
        }
    }

    // 4. 查询任意两点距离
    cout << "1到n的距离: " << dist[1][n] << endl;

    return 0;
}
```

### 🧠 为什么 k 要在最外层？（理解了这点才算学会）
这是 DP 的性质决定的。
Floyd 的本质状态是：$dp[k][i][j]$ 表示 **“只允许经过前 $k$ 个点作为中转站时，从 $i$ 到 $j$ 的最短路径”**。

*   当循环到 `k=1` 时，我们计算的是“只允许经过节点 1 中转”的最短路。
*   当循环到 `k=2` 时，我们在“允许经过节点 1”的基础上，再加入“允许经过节点 2”。
*   如果你把 $k$ 放在里面，就变成了“在这次计算中，我只允许用 $k$ 中转”，但你没有把之前的中转结果累积下来，DP 的状态就断了。

**这就是 Floyd。是不是比 Dijkstra 简单多了？** 😉
你可以找个 $N$ 比较小的图论题试一下，或者自己造个小数据测一下。学会这个，你的图论部分分工具箱就加了一把“万能钥匙”。

## Dijkstra算法
- **“贪心的探险家”
- **一句话原理**：永远只挑**当前离起点最近**的那个点探索，永不回头。
- **核心关联：BFS plus版**
	- BFS用`queue`（FIFO）
	- Dijkstra用`priority_queue`（谁距离短谁先走）
- **适用**：点很多（N<=1e5），没有负权边，求**单源**最短路径。

###  核心修改：队列 (Queue) $\to$ 优先队列 (Priority Queue)

Dijkstra 的算法流程和 BFS **几乎一模一样**，只有两个关键改动：

1.  **容器换了**：
    *   BFS 用 `std::queue`（先进先出）。
    *   Dijkstra 用 `std::priority_queue`（**谁距离短，谁先出**）。
    *   *人设：Dijkstra 是个势利眼，永远只挑当前离起点最近的那个点去拓展。*

2.  **松弛操作 (Relaxation)**：
    *   BFS：只要没访问过，就标记 `visited` 并入队。
    *   Dijkstra：我不管你访没访问过，只要**我发现了一条更近的路**（`new_dist < old_dist`），我就更新你的距离，并把你再次入队。

---

###  Dijkstra 标准模板（抄作业专用）

**前置知识点**：
*   我们需要一个小顶堆（最小值在上面）。
*   C++ 的 `priority_queue` 默认是大顶堆。
*   **变身咒语**：`priority_queue<Type, vector<Type>, greater<Type>>`。
*   我们用 `pair<int, int>` 存 `{距离, 点编号}`，因为 pair 默认先比较第一项（距离），正好符合我们的需求。

```cpp
#include <bits/stdc++.h>
using namespace std;

const int INF = 0x3f3f3f3f; // 无穷大
const int N = 100005;      // 最大点数

// 邻接表：pair<邻居, 边权>
vector<pair<int, int>> adj[N]; 
int dist[N]; // 存储起点到各点的最短距离

void dijkstra(int start) {
    // 1. 初始化
    memset(dist, 0x3f, sizeof(dist)); // 全部设为无穷大
    dist[start] = 0;
    
    // 定义小顶堆：存 {距离, 点编号}
    priority_queue<pair<int, int>, vector<pair<int, int>>, greater<pair<int, int>>> pq;
    
    // 起点入队：距离0，编号start
    pq.push({0, start});
    
    while (!pq.empty()) {
        // 2. 取出当前距离最近的点
        pair<int, int> curr = pq.top();
        pq.pop();
        
        int d = curr.first;  // 当前累计的距离
        int u = curr.second; // 当前点的编号
        
        // 3. 懒惰删除（Lazy Deletion）的关键：
        // 如果从堆里取出的距离 d 比我们要更新的 dist[u] 还要大，
        // 说明这是一个“过期”的旧数据（之前被更优的路径更新过了），直接扔掉。
        if (d > dist[u]) continue;
        
        // 4. 扫描所有邻居（和 BFS 一样）
        for (auto edge : adj[u]) {
            int v = edge.first;  // 邻居编号
            int w = edge.second; // 边权 (u到v的距离)
            
            // 5. 松弛操作：如果经由 u 到 v 更近
            if (dist[u] + w < dist[v]) {
                dist[v] = dist[u] + w; // 更新距离
                pq.push({dist[v], v}); // 把更优的 {新距离, v} 扔进堆里
            }
        }
    }
}

int main() {
    int n, m, s; // n个点, m条边, s是起点
    cin >> n >> m >> s;
    for (int i = 0; i < m; i++) {
        int u, v, w;
        cin >> u >> v >> w;
        adj[u].push_back({v, w}); // 有向图
        // adj[v].push_back({u, w}); // 如果是无向图，这行也要写
    }
    
    dijkstra(s);
    
    // 输出到各点的距离
    for(int i=1; i<=n; i++) cout << dist[i] << " ";
    return 0;
}
```

## Bellman-Ford
思路：对所有边进行N-1次松弛操作，若做了N次松弛还能更新，说明存在**负权环**
O(NM)

Bellman-Ford的队列优化版，每次只把更新了的点加入队列，平均复杂度O(kM)，能处理负权边和负环。但可以被卡到O(NM)
