
在算法竞赛里，快速初始化数组（尤其是大数组）是个高频操作。用对 `memset` 和 `fill` 能让你少写很多循环，同时避免潜在的性能坑。下面我把它们的核心用法和记忆方法讲清楚。

---

### 一、`memset`：按**字节**初始化

头文件：`<cstring>`  
函数原型：`void* memset( void* dest, int ch, std::size_t count );`

**核心特点**：
- 它把目标内存的**每个字节**都设置成 `ch`（`unsigned char`）。
- 所以赋值不是按元素来的，而是按**字节模式**。
- 这决定了它**只能用于每个字节都相同的值**，比如：
  - `0`：字节模式 `0x00`
  - `-1`：补码下所有字节都是 `0xFF`
  - `0x3f`：字节模式 `0x3f`，刚好是我们常用来初始化 `int` 为 `0x3f3f3f3f` 的魔法数。

**常用场景**：
```cpp
int dp[1005];
memset(dp, 0, sizeof(dp));          // 全0
memset(dp, -1, sizeof(dp));         // 全-1
memset(dp, 0x3f, sizeof(dp));       // 全0x3f3f3f3f (常用作“无穷大”)
bool vis[1005];
memset(vis, true, sizeof(vis));     // 全true (true通常就是1)
```
对于 `bool`，`memset(vis, 1, sizeof(vis))` 可行，因为 `true` 存储为非零，1 是常见的实现。

**危险/错误用法**：
```cpp
memset(dp, 1, sizeof(dp)); // 每个字节都是0x01，int数组每个元素变成0x01010101，不是1！
```
**记忆口诀**：`memset` = **memory set**，是内存级别的字节填充，只能填“每个字节都一样”的整数。

---

### 二、`fill`：按**元素**初始化

头文件：`<algorithm>`  
函数原型：`void fill( ForwardIt first, ForwardIt last, const T& value );`

**核心特点**：
- 对 `[first, last)` 内的每个**元素**调用赋值，把 `value` 拷贝给它们。
- **可以用任何值**，不管是 `1`、`3.14` 还是某个结构体。
- 适用于所有 STL 容器和普通数组。

**常用场景**：
```cpp
int a[100];
std::fill(a, a + 100, 42);           // 全填42

std::vector<double> dp(n);
std::fill(dp.begin(), dp.end(), 1e9); // 浮点数初始化为大值

struct Node { int x, y; };
Node nodes[100];
std::fill(nodes, nodes + 100, Node{0, 0}); // 结构体初始化
```

**`fill` 与 `memset` 的关键区别**：

| | `memset` | `fill` |
|---|---|---|
| **操作粒度** | 每个**字节** | 每个**元素** |
| **可填的值** | 仅限字节模式相同的值（0, -1, 0x3f等） | **任意值** |
| **适用容器** | 连续内存（数组、POD结构体） | 所有前向迭代器（数组、vector、list等） |
| **速度** | 通常极快（直接调系统级内存操作） | 稍慢，但差距极小，编译器优化后接近 `memset` |
| **类型安全** | 无（容易用错） | 安全，类型检查 |

**记忆口诀**：`fill` = **填值**，逐个元素塞给你想要的数，通用且安全。

### 三、其他好用的初始化工具

#### 1. `std::fill_n`：指定个数填充
```cpp
std::fill_n(a, n, 0);  // 把 a[0] 到 a[n-1] 赋值为0，类似 memset 但更安全
```
优点是可以直接指定元素个数，不必计算指针范围，但同样只能赋值为正确的元素值。

#### 2. `std::iota`：生成递增序列
头文件：`<numeric>`
```cpp
int a[100];
std::iota(a, a + 100, 1); // a[0]=1, a[1]=2, ..., a[99]=100
```
适用于需要 1~n 的编号时，比循环更简洁。

#### 3. `vector` 自带的构造/赋值
- `vector<int> v(n, 0);`：直接构造 n 个 0，最常用。
- `v.assign(n, value);`：重设大小并填充相同的值。
- `v.resize(n, value);`：改变 size，新增部分用 value 填充。
这些在动态改变大小时比 `fill` 更直接。

---

### 四、记忆与使用策略

1. **能用 `vector` 自带的初始化就不要用循环**：
   ```cpp
   vector<int> dp(n, 0);          // 好
   vector<int> dp; dp.assign(n, 0); // 也行
   ```
2. **需要把整个静态数组重置为 0 或 -1 时**，优先用 `memset`，因为它快且写法简短。
3. **需要填其他值（包括 1、大浮点数、结构体）**，一律用 `fill` 或 `fill_n`。
4. **当数组大小在编译期不定，或者你要填的值是 `int` 的 1，别用 `memset`！** 新手常犯的错误就是把 `memset` 当成万能初始化，结果填出奇怪数字。
5. **用表格帮助记忆**：

| 场景 | 推荐写法 |
|---|---|
| 静态数组，初始全 0 | `memset(a, 0, sizeof(a))` |
| 静态数组，初始全 -1 | `memset(a, -1, sizeof(a))` |
| 静态数组，INF 常量 `0x3f3f3f3f` | `memset(a, 0x3f, sizeof(a))` |
| 静态/动态数组，填任意值 `x` | `fill(a, a+n, x)` 或 `fill(v.begin(), v.end(), x)` |
| 动态数组，初始大小 n 且全填 val | `vector<int> v(n, val)` |
| 需要 1,2,3,... 的序列 | `iota(a, a+n, 1)` |

掌握这两个函数的核心区别，初始化这一步就再也不会出错了。
