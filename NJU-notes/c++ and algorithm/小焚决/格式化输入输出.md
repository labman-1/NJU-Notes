在 P1443 马的遍历这道题里，题目要求输出一个整齐的矩阵，每个数字占 5 个字符宽度，且左对齐。这就是格式化输出的典型应用场景。

---

### 一、 C++ 派：`cout` + `<iomanip>`

你的代码：`cout << left << setw(5) << visited[i][j];`

要使用这些高级格式控制，首先必须引入头文件：
```cpp
#include <iomanip> // io manipulation (IO操纵)
```

#### 1. 代码拆解
*   **`setw(5)`** (Set Width):
    *   **含义**：设定下一次输出内容的“场宽”（Field Width）为 5 个字符。
    *   **效果**：就像给数字画了一个 5 格宽的盒子。如果数字是 `12`（占2格），剩下 3 格会补空格。
    *   **⚠️ 致命坑点**：`setw` 是**一次性**的！它只对紧接着的那个输出有效。下一次输出还要再写一次 `setw`。
*   **`left`** / **`right`**:
    *   **含义**：设定对齐方式。
    *   **效果**：
        *   `left`: 数字靠左，空格补在右边（`12   `）。
        *   `right` (默认): 数字靠右，空格补在左边（`   12`）。
    *   **特点**：它是**持久**的（Sticky）。一旦设置了 `left`，后面的输出都会左对齐，直到你手动改成 `right`。

#### 2. 常用组合拳
*   **左对齐，宽5格**（你的题目要求）：
    ```cpp
    cout << left << setw(5) << x; 
    // 输出: "12   "
    ```
*   **右对齐，宽5格，补0**（比如输出时间 05:00）：
    ```cpp
    cout << right << setfill('0') << setw(2) << 5;
    // 输出: "05"
    ```
*   **保留 2 位小数**：
    ```cpp
    double pi = 3.14159;
    cout << fixed << setprecision(2) << pi;
    // 输出: "3.14"
    ```

### 二、 C语言 派：`printf`

虽然是 C++ 选手，但很多大佬（包括我）在处理**复杂格式**时，更喜欢用 `printf`，因为它写起来更短！

需要头文件：`#include <cstdio>` (即 C 的 stdio.h)

#### 1. 对应写法
你的那行代码，用 `printf` 写是这样的：
```cpp
printf("%-5d", visited[i][j]);
```

是不是短了很多？我们来拆解这个字符串 `%-5d`：
*   **`%`**: 格式控制符的开始。
*   **`-`**: 表示 **左对齐**（如果没有 `-`，默认是右对齐）。
*   **`5`**: 表示 **宽度为 5**。
*   **`d`**: 表示输出的是 **整型 (int)**。

#### 2. `printf` 速查表（背下来，终身受用）

| 符号 | 含义 | 举例 (`x=12`) | 输出结果 (下划线代表空格) |
| :--- | :--- | :--- | :--- |
| `%d` | 普通整数 | `printf("%d", x)` | `12` |
| `%5d` | 宽5，右对齐 | `printf("%5d", x)` | `___12` |
| `%-5d`| **宽5，左对齐** | `printf("%-5d", x)` | `12___` |
| `%05d`| 宽5，补0 | `printf("%05d", x)` | `00012` |
| `%lld`| **Long Long** | `printf("%lld", 1e18)` | `1000000000000000000` |
| `%.2f`| **保留2位小数**| `printf("%.2f", 3.14159)`| `3.14` |

---

### 三、 建议：到底用哪个？

1.  **如果你已经开了 IO 加速** (`ios::sync_with_stdio(false)`):
    *   **混用是大忌！** 千万不要一会 `cout` 一会 `printf`，输出顺序会乱掉。
    *   既然你习惯了 `cout`，那就坚持用 `cout`。记得 `setw` 每次都要写。

2.  **对于浮点数和矩阵输出**：
    *   说实话，`printf` 真的比 `cout` 方便太多了。
    *   `printf("Case #%d: %.2f\n", i, ans);` 一行搞定。
    *   用 `cout` 要写成：`cout << "Case #" << i << ": " << fixed << setprecision(2) << ans << "\n";` 手都写断了。

**结论：**
*   在 P1443 这道题里，你现在的 `cout << left << setw(5) ...` 写法完全没问题，继续保持！
*   如果有朝一日你觉得 `cout` 写格式太累，随时可以切回 `printf`（记得关掉 IO 加速或者只用 `scanf/printf`）。
