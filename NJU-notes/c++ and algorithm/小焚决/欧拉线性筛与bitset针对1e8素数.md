欧拉线性筛是 OI 中最经典的素数算法，而 `bitset` 是 C++ 标准库中被低估的空间优化神器。这两者结合，能让你在处理 **1e7 甚至 1e8** 的素数表时游刃有余。

---

### 一、欧拉线性筛：为什么是线性的？

普通埃筛的问题在于：一个合数会被它的**多个质因数**重复标记。
例如 `12`，在 `p=2` 时被筛一次 (`2*6`)，在 `p=3` 时又被筛一次 (`3*4`)。

欧拉线性筛的核心思想是：**每个合数，只被它的“最小质因数”筛掉**。
这样每个合数在它的“一生”中只被访问一次，因此整体时间复杂度为严格的 **O(N)**。

#### 算法流程（维护两个东西）：
1. `isPrime[x]`：布尔表（初始全为 `true`，0 和 1 为 `false`）。
2. `primes` 数组：按顺序存放已经找到的质数。

**对 `i` 从 `2` 遍历到 `N`：**
- 若 `isPrime[i] == true`，将 `i` 加入 `primes` 列表。
- 然后遍历质数列表中的每个 `p`（即已找出的最小质因数候选）：
  - 如果 `i * p > N`，跳出循环。
  - 将 `isPrime[i * p]` 标记为 `false`（**此时 `p` 一定是 `i * p` 的最小质因数**）。
  - **关键剪枝**：如果 `i % p == 0`，则**立刻跳出内层循环**。

#### 核心代码
```cpp
const int MAXN = 10000000; // 1e7
vector<int> primes;
bitset<MAXN + 1> isPrime;   // 稍后讲解 bitset 妙用

void linear_sieve() {
    isPrime.set();          // 全部置为 1
    isPrime[0] = isPrime[1] = 0;
    for (int i = 2; i <= MAXN; ++i) {
        if (isPrime[i]) {
            primes.push_back(i);
        }
        for (int p : primes) {
            if (i * p > MAXN) break;
            isPrime[i * p] = 0;
            if (i % p == 0) break; // 线性保证的精髓
        }
    }
}
```

### 二、灵魂剪枝：`if (i % p == 0) break;` 凭什么能保证不重不漏？

这是整个线性筛唯一需要记忆和理解的地方。

**原理**：
- 当 `i % p == 0` 时，说明 `p` 是 `i` 的一个质因数。
- 由于我们的质数列表是从小到大排列的，此时 `p` **一定是 `i` 的最小质因数**（因为之前的小质数都没能整除 `i`）。
- 那么对于更大的质数 `p' > p`，合数 `i * p'` 的最小质因数**仍然是 `p`**，而不是 `p'`。

**例子**：`i = 6`（`2 * 3`），质数表 `[2, 3, 5, ...]`
- `p = 2`：`i % 2 == 0`，筛掉 `6*2=12`（最小质因数是 2），然后 **`break`**。
- 如果**不 break**，继续看 `p = 3`，会筛 `6*3=18`。
  但 `18` 的最小质因数是 `2`，它应该在将来 `i = 9` 时用 `p = 2` 去筛掉（`9*2=18`）。现在筛就重复了。

**一句话总结**：
> **`i % p == 0` 意味着 `p` 是 `i` 的最小质因数，因此它也是 `i * p` 的最小质因数。再往后用更大的 `p'` 去筛，就违反了“最小质因数”原则，必须停下来。**

### 三、空间救星：`std::bitset` 的登场

#### 1. `bitset` 是什么？
它是一个**按位存储**的定长二进制数组，每一个元素只占 **1 bit**。

```cpp
bitset<1000000> b;   // 100万个二进制位，仅占约 125 KB 内存
```

#### 2. 对比 `bool` 数组的内存占用

| 类型 | 存储 1e7 个标记所需内存 | 存储 1e8 个标记所需内存 |
| :--- | :--- | :--- |
| `bool isPrime[MAXN]` | ~10 MB | ~100 MB |
| `vector<bool>` | ~1.25 MB | ~12.5 MB |
| `bitset<MAXN>` | ~1.25 MB | ~12.5 MB |

对于 `1e8` 级别的素数筛，`bool` 数组的 **100 MB** 往往接近竞赛的内存限制（通常 256 MB），而 `bitset` 的 **12.5 MB** 则毫无压力。

> **注意**：`vector<bool>` 虽然也被特化为位存储，但它不是容器（不满足容器要求），且在某些操作下较慢。**`bitset` 拥有最极致的位操作性能和固定的内存布局**，是筛法的最佳拍档。

#### 3. `bitset` 常用操作速查（OI 竞赛）

```cpp
bitset<MAXN> bs;

bs.set();        // 全部置为 1
bs.reset();      // 全部置为 0
bs.set(5, 0);    // 把第 5 位置为 0（即 bs[5] = 0）
bs.flip(3);      // 翻转第 3 位

bool val = bs[10];       // 读取第 10 位
int cnt = bs.count();    // 返回 1 的个数

// 位运算（竞赛中常用于集合求交并）
bitset<MAXN> a, b;
a &= b;           // 按位与
a |= b;           // 按位或
a ^= b;           // 按位异或
```

### 四、用 `bitset` 改写欧拉线性筛（高性能模板）

以下代码可以安全地处理 **1e8** 以内的素数表生成（前提是内存允许，且 `int` 范围能存下 `i * p`，1e8 时需注意 `i * p` 可能溢出 `int`，需用 `long long`）。

```cpp
#include <bits/stdc++.h>
using namespace std;

const int MAXN = 100000000; // 1e8
bitset<MAXN + 1> isPrime;
vector<int> primes;

void linear_sieve() {
    isPrime.set();
    isPrime[0] = isPrime[1] = 0;
    for (int i = 2; i <= MAXN; ++i) {
        if (isPrime[i]) {
            primes.push_back(i);
        }
        for (int p : primes) {
            long long nxt = 1LL * i * p; // 防溢出
            if (nxt > MAXN) break;
            isPrime[nxt] = 0;
            if (i % p == 0) break;
        }
    }
}

int main() {
    ios::sync_with_stdio(false);
    cin.tie(0);
    linear_sieve();
    cout << "1e8 以内素数个数: " << primes.size() << endl;
    // 输出：5761455
    return 0;
}
```

### 五、性能与取舍：`bool` 数组 vs `bitset`

| 维度 | `bool isPrime[MAXN]` | `bitset<MAXN>` |
| :--- | :--- | :--- |
| **内存占用** | 1 Byte / 元素 | **1 bit / 元素** |
| **访问速度** | **极快**（直接内存寻址） | 略慢（需位运算提取，但 CPU 位指令优化后差异极小） |
| **写速度** | 快 | 稍慢（同样需位操作） |
| **Cache 友好** | 差（占空间大，易 Miss） | **好**（同样的 Cache 能装下 8 倍数据） |
| **可读性** | `if (a[i])` 自然 | `if (bs[i])` 同样自然 |

**实战建议**：
- **N ≤ 1e7**：随便用 `bool` 数组或 `vector<bool>`，内存无压力。
- **N = 1e8**：**强烈推荐 `bitset`**，它能让你在 256 MB 限制下安全运行，且因为 Cache 命中率高，有时甚至比 `bool` 数组还快。
- **N > 1e8**：`bitset` 内存足够但时间会变长，此时应考虑**分段筛**（例如求 `[L, R]` 区间内素数）。

### 六、竞赛实战中的 `bitset` 妙用延伸

1. **区间筛的辅助数组**：
   当你要筛 `[L, R]`（`R - L ≤ 1e6`，但 `L, R` 可达 `1e12`）时，用 `bitset<1000005>` 来标记区间内的合数，比 `bool` 数组省内存且清零更快（`bs.reset()` 极快）。

2. **常数优化黑科技**：
   `isPrime.flip()` 在某些需要反复反转标记的题里，比手动循环快得多。

3. **容斥原理 / 集合覆盖问题**：
   用 `bitset` 表示一个集合，求交集只需要一个 `&` 操作，C++ 的 `bitset` 在做位运算时会一次性操作一个机器字长（如 64 位），相当于**循环展开了 64 倍**，速度极快。

### 七、总结

- **欧拉线性筛**：`if (i % p == 0) break;` 是唯一需要刻进 DNA 的代码，理解它，你就真正掌握了线性筛。
- **`bitset`**：它是竞赛中处理 **1e7 ~ 1e8** 标记数组时的**空间压缩神器**，写法简单，性能强劲。
- **最佳组合**：`bitset` 做标记数组 + `vector<int>` 存质数，就是目前 OI 圈内处理大范围素数表的最优模板。
