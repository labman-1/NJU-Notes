在 OI/ACM 竞赛中，主要有两种主流存法：
1.  **邻接矩阵**（Adjacency Matrix）—— 简单粗暴，但费空间。
2.  **邻接表**（Adjacency List）—— 省空间，主流首选。
    *   *(还有一种叫“链式前向星”，是邻接表的数组模拟版，等我以后觉得 `vector` 慢了再学，现在先不看)*

### 方法一：邻接矩阵 (Adjacency Matrix) 

这是最直观的方法，就像一个 Excel 表格。
我们开一个二维数组 `g[N][N]`。
*   `g[u][v] = 1` 表示点 $u$ 到点 $v$ 有一条边。
*   `g[u][v] = 0` (或 $\infty$) 表示没有边。
*   如果是有权值的边，那就 `g[u][v] = weight`。
#### 代码实现模板
```cpp
const int N = 1005; // 只能处理小图，N <= 1000 左右
int g[N][N];
int n, m; // n个点，m条边

int main() {
    cin >> n >> m;
    // 初始化：自己到自己是0，到别人是无穷大
    memset(g, 0x3f, sizeof(g)); 
    for(int i = 1; i <= n; i++) g[i][i] = 0;

    for(int i = 0; i < m; i++) {
        int u, v, w;
        cin >> u >> v >> w; // 从 u 到 v 权值为 w
        g[u][v] = w;
        
        // 注意：如果是【无向图】，边是双向的，要加下面这句！
        // g[v][u] = w; 
    }
    return 0;
}
```

#### 评价
*   **优点**：
    *   超级好写。
    *   查询“点 u 和点 v 之间有没有边”非常快，复杂度 $O(1)$。
*   **缺点**：
    *   **空间爆炸**！你需要 $N^2$ 的空间。如果 $N=10000$（1万），$N^2$ 就是 1亿个 int，大概 400MB，直接 **MLE (Memory Limit Exceeded)**。
    *   遍历一个点的所有邻居很慢，必须扫描 1 到 $N$，复杂度 $O(N)$。
*   **适用场景**：
    *   点数 $N \le 500$（比如 Floyd 算法）。
    *   稠密图（边数特别多，$M \approx N^2$）。

### 方法二：邻接表 (Adjacency List) —— 使用 `vector` (强烈推荐)

这是**绝大多数图论题目**的标准写法。
它的思想是：**每个人只记自己的微信好友列表**。
*   点 1 的好友：`[2, 5]`
*   点 2 的好友：`[3]`
*   点 3 的好友：`[]` (孤僻)

我们用 C++ 的动态数组 `vector` 来实现。

#### 1. 存无权图（只存终点）

```cpp
#include <vector>
using namespace std;

const int N = 100005; // 可以开很大，处理 N = 10万 没问题
vector<int> adj[N]; // adj[u] 存的是 u 能直接到达的所有点
int n, m;

int main() {
    cin >> n >> m;
    for(int i = 0; i < m; i++) {
        int u, v;
        cin >> u >> v;
        
        adj[u].push_back(v); // u -> v 有边，把 v 加入 u 的好友列表
        
        //如果是无向图，记得反向也加一下
        // adj[v].push_back(u); 
    }
    return 0;
}
```

#### 2. 存有权图（存终点 + 权值）
如果边有权值（比如距离），`vector` 里只存一个 `int` 就不够了。我们可以存 `struct` 或者 `pair`。

```cpp
struct Edge {
    int to;   // 终点
    int val;  // 权值
};

vector<Edge> adj[N]; // 这是一个 vector 数组

int main() {
    // ... 输入 n, m ...
    for(int i = 0; i < m; i++) {
        int u, v, w;
        cin >> u >> v >> w;
        
        // u -> v，权值 w
        adj[u].push_back({v, w}); 
        
        // 无向图
        // adj[v].push_back({u, w});
    }
}
```

#### 3. 怎么遍历？(BFS/DFS 的基础)
这是邻接表最舒服的地方，只遍历存在的边，不浪费时间。

```cpp
// 比如我们要看点 u 通向哪里
int u = 1;
for (auto edge : adj[u]) {
    // 对于无权图，edge 就是 v
    // cout << "1 连接着 " << edge << endl;

    // 对于有权图，edge 是结构体
    cout << "1 到 " << edge.to << " 的距离是 " << edge.val << endl;
}
```

#### 评价
*   **优点**：
    *   **省空间**！空间复杂度是 $O(N+M)$。存 10万个点、20万条边毫无压力。
    *   **遍历快**！只遍历存在的边。
*   **缺点**：
    *   判断“u 和 v 之间有没有边”比较慢，得去 `adj[u]` 里把列表翻一遍。但在竞赛中很少有题目专门考这个操作。
*   **适用场景**：
    *   **几乎所有图论题**（BFS, DFS, Dijkstra, SPFA, 拓扑排序...）。

---

### 总结一下 

| 特性 | 邻接矩阵 (`g[u][v]`) | 邻接表 (`vector adj[u]`) |
| :--- | :--- | :--- |
| **空间** | $O(N^2)$ (太大了!) | $O(N+M)$ (很省) |
| **查询 u-v 是否相连** | $O(1)$ (极快) | $O(Degree)$ (较慢) |
| **遍历 u 的邻居** | $O(N)$ (慢，要扫一行) | $O(Degree)$ (快，只看好友) |
| **代码难度** | ⭐ | ⭐⭐ (要会 vector) |
| **OI 推荐度** | 10% (仅限小图) | **90% (主力军)** |
