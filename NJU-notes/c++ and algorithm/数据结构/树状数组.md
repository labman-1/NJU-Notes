核心是理解lowbit以及它为树状数组带来的许多美好的性质。树状数组与其树形态的性质 在讲解单点修改之前，先讲解树状数组的一些基本性质，以及其树形态来源，这有助于更好理解树状数组的单点修改。  
我们约定：  
𝑙(𝑥) =𝑥 −lowbit⁡(𝑥) +1。即，𝑙(𝑥) 是 𝑐\[𝑥\] 管辖范围的左端点。 对于任意正整数 𝑥，总能将 𝑥表示成 𝑠 ×2𝑘+1 +2𝑘 的形式，其中 lowbit⁡(𝑥) =2𝑘  
下面「𝑐\[𝑥\] 和 𝑐\[𝑦\]不交指 𝑐\[𝑥\]的管辖范围和 𝑐\[𝑦\] 的管辖范围不相交，即 \[𝑙(𝑥),𝑥\] 和 \[𝑙(𝑦),𝑦\]不相交。「𝑐\[𝑥\]包含于 𝑐\[𝑦\]」等表述同理。

```cpp
#define _CRT_SECURE_NO_WARNINGS
#include<bits/stdc++.h>
using namespace std;
class FenwickTree {
private:
int n;
vector<int>c;
public:
FenwickTree(int size,const vector<int>&a):n(size),c(size+1,0){
    for (int i = 1;i <= n;i++) {
        c[i] += a[i];
        int j = i + lowbit(i);
        if (j <= n) {
            c[j] += c[i];
        }
    }
}
int lowbit(int x) {
    return x & -x;
}
int query(int x) {
    int sum = 0;
    while (x > 0) {
        sum += c[x];
        x -= lowbit(x);
    }
    return sum;
}
int query_range(int x, int y) {
    return query(y) - query(x - 1);
}
void update(int x, int k) {//c[x]+k
    while (x <= n) {
        c[x] += k;
        x += lowbit(x);	
    }
}
};
int main() {
    int n, m;
    cin >> n >> m;
    vector<int>a(n+1);
    for (int i = 1;i <= n;i++) {
        cin >> a[i];
    }
    FenwickTree one(n,a);
    for (int i = 1;i <= m;i++) {
        int q, x, y;
        cin >> q>>x>>y;
        if (q == 1) {
            one.update(x, y);
        }
        else {
            cout<<one.query_range(x, y)<<endl;
        }
    }
    return 0;
}
```

事实上，树状数组（BIT）本质上是一个**维护“动态前缀信息”的工具**，不仅仅用来维护前缀和。

只要题目中隐含了 **“动态修改”** 和 **“前缀统计”** 这两个特征，就可以考虑用树状数组。我们可以把它的应用场景归纳为以下 **三大模型**：

---

### 模型一：位置维度的维护（基础用法）
**场景**：下标 `i` 就是数组的物理位置。
**核心功能**：单点修改 + 区间求和。

*   **典型题目**：
    *   给你一个数组，经常要把第 $i$ 个数加上 $k$。
    *   经常要询问区间 $[L, R]$ 的和。
*   **判断标准**：
    *   数据是**动态**变化的（如果是静态的，直接用前缀和数组 $O(1)$ 查询更好）。
    *   操作满足**结合律**和**可逆性**（主要是**求和**，也可以是异或）。
    *   *注：求区间最大/最小值（RMQ）通常用线段树，因为 MAX/MIN 不好“减回去”。*

---

### 模型二：数值维度的维护（权值树状数组 / 频次桶）
**场景**：下标 `i` 代表**数值**（或者离散化后的排名）。
**核心功能**：查询“比 $x$ 小的数有多少个”、“第 $k$ 小的数是谁”。

这是树状数组最骚的操作，也就是你刚学的**求逆序对**的思路。在这里，`tree[i]` 存的不是 $a[i]$ 的值，而是数值 $i$ **出现的次数**。

*   **典型题目**：
    *   **逆序对**：求在他前面比他大的数。
    *   **动态排名**：在一个不断插入/删除数字的流中，问某个数目前排第几？
    *   **求第 K 小**：利用树状数组 + 二分查找（或者倍增），可以在 $O(\log^2 N)$ 或 $O(\log N)$ 找到第 $K$ 小的数。
*   **判断标准**：
    *   题目只关心**相对大小**关系。
    *   需要动态统计**有多少个数满足...。

---

### 模型三：配合“差分”思想（进阶用法）
**场景**：把树状数组建在**差分数组**上。
**核心功能**：支持 **区间修改** + **单点查询**。

我们知道，普通树状数组只支持“单点改、区间查”。如果题目让你“把区间 $[L, R]$ 里的每个数都加上 $k$”，然后问“第 $i$ 个数现在是多少”，怎么办？

*   **原理**：
    *   定义差分数组 $D[i] = A[i] - A[i-1]$。
    *   根据数学性质：$A[x] = \sum_{i=1}^x D[i]$（原数组的值 = 差分数组的前缀和）。
    *   **区间修改** $[L, R]$ 变成：$D[L] += k$，$D[R+1] -= k$（把区间操作变成了两个单点操作！）。
    *   **单点查询** $A[x]$ 变成：求 $D$ 的前缀和。
*   **结论**：用树状数组维护差分数组 $D$，就能完美解决这个问题。

---

### 💡 灵魂拷问：为什么不用线段树？

你会发现，凡是树状数组能干的，线段树（Segment Tree）都能干，而且线段树还能干更多（比如区间乘法、区间最值、区间赋值）。

**那为什么还要学树状数组？**

1.  **好写**：树状数组核心代码就 5 行，线段树起码 50 行。考场上写线段树容易手滑写挂，调试火葬场。
2.  **常数小**：树状数组跑得飞快，空间也省（$O(N)$ vs 线段树 $O(4N)$）。
3.  **更优雅**：在处理**逆序对**、**二维偏序**（比如求 $(x_i < x_j, y_i < y_j)$ 的点对数量）这类问题时，树状数组的“权值桶”写法是标准解法。

### 总结：什么时候用？

当你看到题目满足以下 **3 个特征** 时，直接掏出树状数组：

1.  **有动态更新**（Dynamic Update）：数据不是死的，会变。
2.  **涉及前缀统计**（Prefix Statistic）：求前 $k$ 个的和、前 $k$ 个的计数。
3.  **操作可逆/可累加**：主要是加减法。

**下意识反应链：**
*   看到“单点修，区间查” $\to$ **树状数组**。
*   看到“逆序对”、“求比它小的数有多少” $\to$ **离散化 + 权值树状数组**。
*   看到“区间修，单点查” $\to$ **差分 + 树状数组**。
*   看到“区间最值 (RMQ)” $\to$ **线段树** 或 **ST表**（如果是静态）。

## 时间戳优化  
对付多组数据很常见的技巧。若每次输入新数据都暴力清空树状数组，就可能会造成超时。因此使用 tag 标记，存储当前节点上次使用时间（即最近一次是被第几组数据使用）。每次操作时判断这个位置 tag中的时间和当前时间是否相同，就可以判断这个位置应该是 0还是数组内的值。
