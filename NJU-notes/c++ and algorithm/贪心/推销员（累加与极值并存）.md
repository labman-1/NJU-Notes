# [P2672 [NOIP 2015 普及组] 推销员 - 洛谷](https://www.luogu.com.cn/problem/P2672)
核心优化：把动态变化的边界，用预处理数组固化下来

你展现了一个**成熟 OIer 必须具备的核心素养**：
1. **敏锐的复杂度雷达**：立刻反应出 $10^5$ 的数据量拒绝 $O(N^2)$，把悬崖勒马的直觉刻在了骨子里。
2. **精准的矛盾提炼**：你发现了这道题最恶心的地方——疲劳值是“**累加**”的（希望每次挑大的），而路程是“**取最大值且不重复**”的（会产生覆盖效应）。

但是，你之所以在选完第一个之后陷入了“左边排序、右边重算”的 $O(N^2)$ 泥潭，是因为你掉进了一个贪心题最经典的**思维陷阱**——**“增量构造法（走一步看一步）”**。

你总是在想：“我已经选了 $x$ 个，第 $x+1$ 个我该怎么挑？”
因为最大距离 $S$ 这个边界是动态变化的，你走一步看一步，边界就不停地在变，维护它的成本自然就飙升到了 $O(N^2)$。

来，学姐教你 OI 竞赛中做这种“复合贪心题”的**降维打击思维：从“基准线（Baseline）”出发，寻找“微调（Swap）”的可能。**

请准备好草稿纸，回答学姐的三个灵魂拷问：

---

### 第一问：建立绝对的“基准线” 📏

我们先做个梦。假设这道题**没有路程的疲劳值**（即不管多远，路程疲劳都算 0），只要你挑 $X$ 个住户，使得推销疲劳值 $\sum A_i$ 最大。
**学姐提问：**
1. 这个时候你怎么挑？是不是闭着眼睛把所有的 $A_i$ **从大到小排个序**，然后无脑拿前 $X$ 个就行了？
2. 假设我们现在就按这个最暴力的策略，拿了 $A_i$ 最大的前 $X$ 个。这前 $X$ 个住户里，必然有一个离门卫室最远的距离，我们叫它 $MaxS$。
   此时，我们获得的总疲劳值是：`前 X 个 A 的和 + 2 * MaxS`。

**我们把这个组合称为“贪心基准解”。** 接下来，我们只要思考：**有没有可能打破这个基准解？**

---

### 第二问：打破基准的“唯一理由” 🔨

现在我们手里攥着 $A_i$ 最大的前 $X$ 个住户。
没被我们选中的那些住户（也就是排在 $X$ 名之后的），它们的 $A_i$ 肯定都比较小。

**学姐提问：**
1. 既然没选中的人 $A_i$ 都比较小，那我们**凭什么**要把手里的某个人踢掉，换一个没选中的人进来？
2. 唯一的理由是不是：没选中的那个人，虽然推销疲劳 $A_i$ 不如我手里的，但他的**位置特别远**，远到他带来的**路程增量 ($2 \times S$)**，足以弥补他 $A_i$ 的劣势，甚至还能让我大赚一笔？

---

### 第三问：精妙的“微调（Swap）”法则 ⚖️

顺着第二问的思路，如果我们要为了“更远的路程”去做一次替换（踢掉一个已选的，换进一个未选的）。

**学姐提问：**
1. **该踢谁？**
   在已经选好的前 $X$ 个住户里，为了让我的损失最小，我应该踢掉 $A_i$ 最大的，还是 $A_i$ 最小的？（也就是排在第几名的人？）
2. **该换谁进来？**
   在所有**没被选中的、且距离大于 $MaxS$ 的住户里**，我应该挑一个什么样的住户？是挑 $S$ 最大的？还是挑 $A$ 最大的？还是挑某个**组合属性**最大的？（提示：看题目计算总疲劳的公式）。
3. **最核心的顿悟：需要换两个人吗？**
   由于路程疲劳只看**最远**的那一个（取 max，不叠加）。只要我们换进来一个极远的住户，路程的红利就**已经吃满了**！再换第二个远处的住户进来，还能产生额外的路程红利吗？
   既然没有额外的路程红利，而在 $A_i$ 上我们又是亏的，那我们还有必要换第二个人吗？

---

### 💡 学姐的思维升华总结

在 OI 中，当你发现“累加属性”和“极值属性”起冲突，导致走一步看一步很难维护时，请立刻切换视角：

1. **先满足“累加属性”**（按 $A_i$ 从大到小选前 $X$ 个，得到 `Sum + 2 * MaxS`）。
2. **再用“极值属性”去微调**（踢掉一个 $A_i$ 最小的，换进一个能把极大值撑得更大的，比较这两种情况谁更优）。

当你把第三问想通了，你会发现，你只需要知道：
* 前 $X$ 个 $A_i$ 的和。
* 前 $X$ 个里最大的 $S$。
* 第 $X$ 名之后的住户里，`2*S + A` 的最大值。

这些东西，是不是全都可以用 **前缀和** 和 **前/后缀最大值数组**，在 $O(N)$ 的时间里预处理出来？！
```c++
#include<bits/stdc++.h>
using namespace std;
int n;
struct household {
	int num,promote = 0;
	bool operator<(const household& a)const {
		return this->promote < a.promote;
	}
	bool operator>(const household& a)const {
		return this->promote > a.promote;
	}
	friend bool greater(const household& a, const household& b);
}residents[100005];
// 修改greater为函数对象，避免与标准库std::greater冲突
struct HouseholdGreater {
    bool operator()(const household& a, const household& b) const {
        if (a.promote == b.promote) return a.num > b.num;
        return a.promote > b.promote;
    }
};
long long tiredness = 0;
long long  prePromote[100005];
long long  preDistance[100005];
long long suffix[100005];
long long distances[100005]; 
int main() {
	ios::sync_with_stdio(false);cin.tie(0);
	cin >> n;
	for (int i = 1;i <= n;i++) {
		cin >> distances[i]; 
	}
	for (int i = 1;i <= n;i++) {
		cin >> residents[i].promote;
		residents[i].num = i;
	}
	sort(residents + 1, residents + n + 1, HouseholdGreater());
	
	//预处理，收集promote前缀和及前x个里面最大的s，以及2*s+a的后缀和
	for (int i = 1;i <= n;i++) {
		prePromote[i] = prePromote[i - 1] + residents[i].promote;
		preDistance[i] = max(preDistance[i - 1], distances[residents[i].num]);
	}
	//suffix表示排序后，后i个2*s+a的最大值
	suffix[n] = 2 * distances[residents[n].num] + residents[n].promote;
	for (int i = n-1;i >=1;i--) {
		suffix[i] = max(suffix[i+1], 2 * distances[residents[i].num] + residents[i].promote);
	}
	for (int i = 1;i <= n;i++) {
		long long tirednessa = prePromote[i]+2*preDistance[i];
		long long tirednessb = prePromote[i - 1] + suffix[i];
		tiredness = max(tirednessa, tirednessb);
		cout << tiredness<<'\n';
	}
}
```
