# 概念题
## 1.声明式程序设计与命令式程序设计的主要区别是什么？请简要说明。
区别在于重点在“做什么”还是“怎么做”。
声明式程序设计关注目标结果，可以忽略底层的执行细节，只需要声明我要干什么。SQL体现了声明式程序设计。
命令式程序设计关注解决问题的具体步骤和控制流，需要告诉程序每一步该做什么，比如C、C++、Java等。
## 2.函数式程序设计有哪些基本特征
函数式编程属于声明式编程的一个分支。
基本特征如下：
- **函数是“一等公民” ：** 函数与普通数据（如整数、字符串）拥有同等地位。函数可以被赋值给变量，可以作为参数传递给其他函数，也可以作为其他函数的返回值（即Higher order function）。
2.  纯函数： 
    *   **无副作用：** 函数在执行过程中，绝不修改外部的全局变量、不修改传入的参数对象、不进行未预期的 I/O 操作。
    *   **引用透明性：** 给定相同的输入，永远返回完全相同的输出。
3.  **数据的不可变性：** 变量一旦被创建并赋值，其在内存中的状态就绝不能被修改。如果需要改变数据，只能通过创建一个包含新值的新数据结构来实现（这消除了多线程并发时的锁竞态问题）。
4.  **倾向于使用递归而非循环：** 因为传统的 `for/while` 循环本质上依赖于一个不断突变的迭代器变量（如 `i++`），而函数式编程拒绝状态突变，所以通常利用递归来实现迭代逻辑。

## 3. 与一般递归实现相比，尾递归调用有什么优点？
*   **一般递归的不足：**
    在一般递归中，递归调用之后通常还有其他操作（比如 `return n * factorial(n-1)`，必须要等 `factorial(n-1)` 返回结果后，才能执行乘 `n` 的操作）。这导致系统必须为每一次函数调用在内存中保留一个**栈帧**来保存局部变量和返回地址。如果递归深度为 $N$，空间复杂度就是 $O(N)$，极易导致**栈溢出**。

    尾递归：函数的最后一步操作，是仅仅调用自身（不能包含任何其他计算动作）。

*   **尾递归的优点（核心在编译器优化）：**
    由于尾递归调用的返回值就是当前函数的返回值，当前函数在发起递归调用后，已经没有任何需要执行的后续逻辑，原有的局部变量也都不再需要了。
    *   **尾调用优化 ：** 现代编译器或解释器（如 Scheme, C++开启 `-O2` 优化时）识别到尾递归后，会直接复用当前的栈帧，而不是压入新的栈帧。
    *   **性能更优：** 它将原本 $O(N)$ 的空间复杂度瞬间降维打击到了 **$O(1)$**，彻底消除了栈溢出的风险，并且避免了频繁分配/释放栈内存的 CPU 开销。在底层机器码执行上，尾递归会被编译器优化成一个极其高效的 `goto` 或普通的循环跳跃。
# 编程题
## 1. 青蛙跳台阶问题

```c++
#include <iostream>

// 1. 一般递归 
int f_recursive(int n) {
    if (n == 1) return 1;
    if (n == 2) return 2;
    return f_recursive(n - 1) + f_recursive(n - 2);
}

// 2. 尾递归 
// a 代表 f(k), b 代表 f(k+1)，随着 n 递减，状态向前推进
int f_tail(int n, int a = 1, int b = 2) {
    if (n == 1) return a;
    if (n == 2) return b;
    // 尾调用：最后一步仅调用自身，不包含任何外部计算
    return f_tail(n - 1, b, a + b); 
}

// 3. 非递归 / 迭代 
int f_iterative(int n) {
    if (n == 1) return 1;
    if (n == 2) return 2;
    int a = 1, b = 2, c = 0;
    for (int i = 3; i <= n; ++i) {
        c = a + b; 
        a = b;
        b = c;
    }
    return b;
}

int main() {
    std::cout << " n | 普通递归 | 尾递归 | 非递归\n";
    std::cout << "---|----------|--------|--------\n";
    for (int n = 3; n <= 10; ++n) {
        std::cout << " " << n << " |    " 
                  << f_recursive(n) << "    |   " 
                  << f_tail(n) << "   |   " 
                  << f_iterative(n) << "\n";
    }
    return 0;
}
```
