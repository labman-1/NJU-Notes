### 1. 声明式程序设计与命令式程序设计的主要区别是什么？请简要说明。

**核心区别在于：关注“做什么（What）”还是关注“怎么做（How）”。**

*   **命令式程序设计 (Imperative Programming):**
    *   **关注点：** 关注解决问题的**具体步骤和控制流（How）**。
    *   **系统视角：** 程序是由一系列改变机器状态（内存）的指令组成的。开发者需要显式地管理变量的赋值、循环（for/while）和状态的流转。
    *   **典型语言：** C, C++, Java 等传统代码。
    *   *举例：* 想要一个数组的所有偶数，你需要写一个 `for` 循环，定义一个 `index`，判断取模，然后 `push` 到新数组里。

*   **声明式程序设计 (Declarative Programming):**
    *   **关注点：** 关注问题的**逻辑本质和目标结果（What）**，将底层的执行细节（如循环、状态机）交给语言的编译器或解释器去处理。
    *   **系统视角：** 弱化或消除了显式的状态突变（Mutation）和控制流。代码更像是一组数学等式或规则的集合。
    *   **典型语言：** SQL（数据库查询）, HTML（UI构建）, 以及所有函数式语言。
    *   *举例：* 同样取偶数，你只需写 `array.filter(is_even)`，你只声明了“我要过滤出偶数”的规则，而不在乎底层是如何遍历内存的。

### 3. 与一般递归实现相比，尾递归调用有什么优点？

这是一个直接触及**计算机底层调用栈（Call Stack）**的硬核问题。

*   **一般递归的痛点：**
    在一般递归中，递归调用之后通常还有其他操作（比如 `return n * factorial(n-1)`，必须要等 `factorial(n-1)` 返回结果后，才能执行乘 `n` 的操作）。这导致系统必须为每一次函数调用在内存中保留一个**栈帧（Stack Frame）**来保存局部变量和返回地址。如果递归深度为 $N$，空间复杂度就是 $O(N)$，极易导致**栈溢出（Stack Overflow）**。

*   **尾递归的定义：**
    尾递归（Tail Recursion）是指：**函数的最后一步操作，是仅仅调用自身（不能包含任何其他计算动作）**。
    例如：`return tail_factorial(n-1, acc * n)`。

*   **尾递归的优点（核心在编译器优化）：**
    由于尾递归调用的返回值就是当前函数的返回值，当前函数在发起递归调用后，**已经没有任何需要执行的后续逻辑，原有的局部变量也都不再需要了。**
    *   **尾调用优化 (Tail Call Optimization, TCO)：** 现代编译器或解释器（如 Scheme, C++开启 `-O2` 优化时）识别到尾递归后，会**直接复用当前的栈帧**，而不是压入新的栈帧。
    *   **性能飞跃：** 它将原本 $O(N)$ 的空间复杂度瞬间降维打击到了 **$O(1)$**，彻底消除了栈溢出的风险，并且避免了频繁分配/释放栈内存的 CPU 开销。在底层机器码执行上，尾递归会被编译器优化成一个极其高效的 `goto` 或普通的循环（Iteration）跳跃。

**总结：** 尾递归在代码书写上保留了函数式递归的数学优雅性，在底层硬件执行上却拥有媲美 `while` 循环的极致空间效率。
