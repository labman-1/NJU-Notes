### 1\. “0-1-N-Max” 检查法（黄金法则）  

无论题目是什么（链表、树、数组、字符串），都要检查这四种维度：  
●0 (Empty/Null):  
○输入是 Link.empty 吗？  
○输入是 None 吗？  
○输入是 \[\] 或 "" 吗？  
○你的这次痛点就在这里。 永远问自己：如果什么都没有，我的代码会不会 AttributeError？  
●1 (Singleton):  
○链表只有一个节点 Link(1)。  
○循环条件 while lnk.rest is not Link.empty 对单节点是否成立？  
○递归的 Base Case 是否覆盖了剩下一个的情况？  
●N (General/Middle):  
○一般的、正常的情况（比如题目给的 Example）。  
●Max (Limits/Tail):  
○问题出现在末尾怎么办？（例如：Link(1, Link(2, Link(2)))，重复在最后）。  
○全部重复怎么办？（Link(1, Link(1, Link(1)))）。  
○完全不重复怎么办？  
### 2\. 对拍技巧 (Fuzz Testing / Property-based Testing)  

当你实在想不出哪里错了，而你又会写 Python 时，可以自己写一个\*\*“对拍器”\*\*。  
思路：  
●写一个暴力但绝对正确的方法（比如把链表转成 Python List，用 set 去重，再转回链表）。  
●写一个随机数据生成器。  
●循环 1000 次，比较你的 remove\_duplicates 和“暴力法”的结果。

### 3\. 代码覆盖率思维 (White-box Thinking)  

看着你的代码，问自己：  
●if: 有没有测试用例让这个 if 为 True？有没有用例让它为 False？  
●while: 有没有用例让这个循环一次都不执行？（这就是 Link.empty 的情况）
