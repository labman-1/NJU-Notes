# 完全背包 疯狂采药

[https://www.luogu.com.cn/problem/P1616](https://www.luogu.com.cn/problem/P1616#ide)  

```cpp
#include <bits/stdc++.h>
#define T 10000005
#define M 10005
long long d[T];
int times[M], value[M];
using namespace std;
int main() {
	ios::sync_with_stdio(false);
	cin.tie(0);
	int t, m;
	cin >> t >> m;
	for (int i = 1; i <= m; i++) cin >> times[i] >> value[i];
	for (int i = 1;i <= m;i++) {
		for (int j = times[i];j <=t;j++) {
			d[j] = max(d[j - times[i]] + value[i], d[j]);
		}
	}
	cout<<d[t];
	return 0;
}
```

●0/1 背包（倒序）：计算 d\[j\] 时，引用 d\[j - times\[i\]\]。因为是倒序，左边的 d 还是上一轮（没放过物品 i）的状态。→ 要么不放，要么放 1 个。  
●完全背包（正序）：计算 d\[j\] 时，d\[j - times\[i\]\] 已经在这一轮被更新过了（可能已经放进去了 1 个物品 i）。  
○如果 d\[j - times\[i\]\] 里已经有 1 个 i，那你再加一个，就变成了 2 个。  
○如果后面 d\[k\] 又引用了 d\[j\]，那就变成了 3 个……→实现了“只要放得下，想放几个放几个”。  
代码极其优美，只有两个字符的区别（n vs 1，++ vs \--），背后的数学逻辑却完全相反。
