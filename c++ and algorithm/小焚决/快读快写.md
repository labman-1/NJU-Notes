# 快读快写

在C++中，特别是处理大量数据时（例如数据量级达到1e5或1e6），输入输出（I/O）操作的效率会显著影响程序的整体性能，甚至可能导致时间超限（TLE）。理解“流同步加速”并掌握相关优化技巧至关重要。  
### 🔄 理解流同步加速  

C++的std::cin和std::cout等I/O流为了与C语言的scanf和printf等标准I/O函数安全兼容，默认情况下会与它们保持同步。这意味着两者的操作可以按预期顺序交错进行，但为此付出了性能代价。  
“流同步加速”的核心在于解除这种同步和绑定关系，让C++流可以独立、高效地工作。这主要通过两个关键操作实现：  
1取消同步 (std::ios::sync\_with\_stdio(false))调用此函数后，C++标准流不再与C标准流同步，从而允许它们使用独立的缓冲区，这可以显著减少I/O操作的开销。重要注意事项：一旦关闭同步，绝对不要在程序中混用cin/cout和scanf/printf等C风格的I/O方式，否则可能导致输出顺序混乱或数据丢失。  
2解除绑定 (std::cin.tie(nullptr))默认情况下，std::cin与std::cout是绑定在一起的。这意味着每次通过std::cin进行输入操作之前，系统都会先自动刷新（清空）std::cout的缓冲区，以确保在等待用户输入前所有输出都已显示。这会增加不必要的I/O负担。使用std::cin.tie(nullptr)可以解除这种绑定，进一步提升效率。解除绑定后，如果需要确保提示信息在输入前显示（例如提示"Please enter a number: "），可能需要手动使用std::flush或std::endl来刷新输出缓冲区。  
通常，这两个优化会一起使用，放在main函数的开头：  
```cpp
#include <iostream>
int main() {
    std::ios::sync_with_stdio(false);
    std::cin.tie(0);
    // ... 后续代码
    return 0;
}
```
### ⚡ 进阶I/O优化技巧  

除了上述两个基本技巧，还有更多方法可以进一步提升I/O效率。  
●选择 \\n而非 std::endl在输出换行时，使用'\\n'而不是std::endl。因为std::endl不仅输出换行符，还会强制刷新输出缓冲区，这种频繁的刷新操作会降低性能。  
●使用C风格I/O如果问题允许，直接使用C语言的scanf和printf。它们在默认情况下通常就比未优化的C++流要快。但在关闭C++流同步后，就不要再与C++流混用了。  
●手动实现“快读”/“快写”对于极端追求速度的场景（如算法竞赛），可以手动实现基于getchar/putchar的输入输出函数。其原理是直接读取或写入单个字符，然后组合成整数，这通常比标准库函数更快。下面是一个简单的快读函数示例：  
```cpp
int read() {
    int x = 0, w = 1;
    char ch = getchar();
    while (ch < '0' || ch > '9') {
        if (ch == '-') w = -1;
        ch = getchar();
    }
    while (ch >= '0' && ch <= '9') {
        x = x * 10 + (ch - '0');
        ch = getchar();
    }
    return x * w;
}
```
类似地，也可以实现基于putchar的快写函数。  
●使用大容量缓冲区（fread/fwrite）对于海量数据，可以使用fread一次性将大量数据读入自定义内存缓冲区，然后进行处理；输出时则使用fwrite一次性写入。这减少了系统调用的次数，效率非常高。  
### 💎 实践建议与注意事项  

总结一下关键点，并给出一些实用建议：  

| 优化技巧 | 主要作用 | 注意事项 |
| --- | --- | --- |
| sync_with_stdio(false) | 禁用C++与C流同步，大幅提升速度 | 此后禁止混用cin/cout与scanf/printf |
| cin.tie(nullptr) | 解除cin与cout绑定，减少不必要刷新 | 需在第一次I/O操作前调用 |
| 使用\n替代endl | 避免不必要的缓冲区刷新 | 在需要立即显示输出时才用endl或flush |
| C风格I/O (scanf/printf) | 默认情况下速度有优势 | 格式控制需谨慎 |
| 手动快读/快写 | 极致的I/O速度 | 代码复杂度增加，通常针对特定数据类型 |

●调用时机：优化语句务必放在main函数的最开头，在任何实际的I/O操作之前。  
●编译器优化：在编译时使用编译器的高优化级别（如GCC的\-O2或\-O3选项）也能提升程序整体性能，包括I/O相关部分的效率。  
●权衡之道：在大多数应用场景下，使用sync\_with\_stdio(false)和cin.tie(nullptr)就足以满足性能需求，同时能保持代码的简洁和C++的风格。手动快读快写通常只在性能瓶颈非常明显、且数据量极大的特定场景（如算法竞赛）中使用。
