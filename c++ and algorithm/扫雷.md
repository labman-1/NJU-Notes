### 2\. 竞赛中的“数组” vs “Vector”  

你使用了 vector <vector<char>>。  
vector <vector<char>> a(n + 2, vector<char>(m + 2, '?'));
在工程开发（写软件）时，这是极好的习惯（动态分配、安全）。 但在 OI/ACM 竞赛中，对于这种数据范围固定的网格题（N,M≤100），我们更倾向于直接开全局静态数组。  
为什么？  
●打字快： `char a[105] ; 比那一长串 vector 定义要好写得多，考试每一秒都很宝贵。  
●调试方便：在 GDB 或 IDE 调试器里，静态数组的数据查看通常比嵌套 vector 更直观。  
●常数优化：Vector 的动态内存分配和二次寻址会有微小的性能开销（虽然在这题完全忽略不计），但在某些卡常数（Time Limit 很紧）的题目里，静态数组更稳。  
●防爆栈：如果是一个巨大的数组（比如  10000×10000），开在 main 函数里（栈内存）会直接导致 Stack Overflow 报错，必须开在全局（全局数据区）或者用 vector（堆内存）。既然要考虑位置，不如养成大数组开全局的习惯。  
建议写法：  

```cpp
const int MAXN = 105;
char a[MAXN][MAXN]; // 定义在 main 外面，自动初始化为 0 (null)，或者手动初始化
```

### 3. 关于方向数组（Direction Array）  

我之前提到的“更通用的技巧”，其实是指 方向数组。虽然你用的双重循环对于“周围一圈”很完美，但如果题目变成“马走日”或者“上下左右四方向”，双重循环就不好使了。  
进阶写法展示（你可以存下来以后用）：  
```cpp
// 定义8个方向的偏移量
int dx[] = {-1, -1, -1, 0, 0, 1, 1, 1};
int dy[] = {-1, 0, 1, -1, 1, -1, 0, 1};

// 在循环里：
for (int k = 0; k < 8;  k++) {
    int nx = i + dx[k];
    int ny = j + dy[k];
    if (a[nx][ny] == '*') cnt++;
}
```

这种写法在以后写 BFS（广度优先搜索） 或 DFS 迷宫类问题时是标配，建议尽早熟悉。  
### 4. endl 也是个小刺客  

`cout << endl;` endl 不仅换行，还会强制刷新缓冲区。在输出量巨大的题目中（比如输出 100万行），频繁使用 endl 会导致严重的 I/O 耗时，甚至导致 TLE。 习惯： 尽量使用 '\\n' 来换行。  
```cpp
cout << a[i][j]; // 这一行没问题
cout << '\n';    // 每一行结束用 \n
```
